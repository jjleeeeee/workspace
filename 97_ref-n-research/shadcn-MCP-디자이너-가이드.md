# shadcn MCP - 디자이너 관점 정리

## shadcn MCP란?

한 줄 요약: **AI 어시스턴트(Claude, Cursor 등)에서 UI 컴포넌트를 자연어로 검색·설치할 수 있게 해주는 다리**

```
개발자 → "랜딩 페이지 만들어줘"
  → AI가 shadcn MCP 통해 컴포넌트 레지스트리 조회
  → hero, features, testimonials 블록 자동 설치 + 코드 생성
```

### 핵심 기능
- **컴포넌트 목록 조회** - 어떤 컴포넌트/블록/템플릿이 있는지 리스트
- **자연어 설치** - "버튼, 다이얼로그 추가해줘" 하면 자동 설치
- **프라이빗 레지스트리 지원** - 회사 자체 컴포넌트 라이브러리를 MCP에 연결 가능
- 지원 클라이언트: Claude Code, Cursor, VS Code, Codex

---

## 배경 - 개발자분이 공유한 방향성

```
[기존 시도] 완전 AI 자동화
  → 너무 변수가 많음, 컨트롤 불가 ("무한면 주사위 굴리기")

[현재 방향] UI 빌더
  → 선택지를 제한함 ("주사위 면 수를 줄임")
  → 기획/디자이너/개발자가 클릭으로 페이지 구성
  → 구성 결과를 DSL(구조화된 데이터)로 추출
  → DSL을 AI한테 넘기면 코드로 구현
     또는 DSL 렌더러가 직접 화면으로 표시
```

쉽게 말하면 **Webflow, Framer 같은 노코드 빌더의 기업용 버전**. 차이점은 출력이 "렌더링된 화면"이 아니라 "DSL"이라는 것.

> 페이지별 DSL이 많이 쌓일수록 → 다음 자동화도 쉬워진다

## 디자이너가 해야 할 일

### 1. shadcn MCP 직접 써보기 (지금 당장)
Claude Code에서 실제로 체험.

```
Claude Code 열고 → shadcn MCP 설정 → 
"연락처 폼 컴포넌트로 만들어줘" 입력
```

**목적:** AI가 컴포넌트를 어떻게 조합하는지 패턴을 눈으로 확인 → UI 빌더 설계에 반영

### 2. 블록 목록 정의
UI 빌더의 핵심은 **선택지 목록(컴포넌트 레지스트리)**. 디자이너 역할:

- 자사 서비스에서 반복되는 UI 패턴 목록화 (Hero / Feature카드 / 폼 / 테이블 / 모달 등)
- 각 블록의 **속성(Props) 정의** — 텍스트, 색상, 레이아웃 옵션
- Figma 컴포넌트 정리 → shadcn 스펙에 맞게 명세 작성

### 3. UI 빌더 UX 설계
"기획/디자이너/개발자가 클릭으로 구성하는 화면"은 디자이너가 설계해야 함.

- 왼쪽 패널: 블록 목록
- 가운데: 캔버스 (드래그앤드롭)
- 오른쪽: 속성 편집
- 내보내기: DSL JSON / 코드

### 4. DSL 구조 협의
개발자와 함께 "이 화면을 어떻게 데이터로 표현할 것인가" 결정. 예시:

```json
{
  "page": "landing",
  "sections": [
    { "type": "hero", "title": "제목", "cta": "시작하기" },
    { "type": "features", "items": [...] }
  ]
}
```

필요한 속성이 빠지지 않았는지 디자이너 관점에서 검토하고 협의.
