# 방법론 회고 및 향후 체크리스트

---

## 이번 제작 방식의 강점

### 양극 페어링 전략
Yuki(20대 Advocate/외향)와 Satomi(40대 Silent Watcher/내향)를 일본 팬덤 스펙트럼의 양 끝으로 배치.
- 하나의 UX 결정이 두 극단 모두에게 어떻게 달리 작동하는지 포착 가능
- 디자이너가 "중간" 유저를 상상하는 것보다 명확한 의사결정 기준 제공

### 다층 리서치 교차 검증
핵심 행동(누이카츠, 모녀 입덕, LINE 오픈채팅, 코우칸)은 3~4개 소스에서 일관되게 등장.
단일 소스 의존 없이 공통 패턴 확인.

### 엔소나 방법론 이식
평가 레이어(rubric-v0, judge.md)가 강연 방법론과 정렬됨.
외압 저항(Assertive Stance) 원칙이 두 퍼소나 대화 규칙에 명시적으로 이식됨.

## 이번 제작 방식의 약점

### 정량-정성 통합 불균등

| 퍼소나 | 정량 데이터 반영 |
|-------|--------------|
| Satomi | ✅ `insight-data-table.md` → Silent Watcher 정체성 직접 결정 |
| Yuki | ❌ 정량 데이터 의도적 미반영 (양극 페어링 목적이지만 명시적 근거 없음) |

→ 향후 Yuki 정량 근거 보완 필요: 일본 내 "고관여 능동 팬" 비율 데이터 확보하면 Yuki의 소수 대표성 명시 가능.

### CREAT Q4 과부하

`CREAT-팬 퍼소나 제작 질문` Q4(앱 생태계) 단일 질문이 두 퍼소나의 앱 생태계 **전체**를 감당.
메루카리·트레포탈·아이돌챔프·KDOL·치어링애드·생일JAPAN·오시바나 — 7개 앱이 Q4 하나에서 유래.

리스크: Q4 답변자의 개인 편향이 전체 앱 설정에 반영될 수 있음.
보완: 같은 앱 목록을 독립적 리서치(실사용 통계, 앱스토어 순위 등)로 교차 검증.

### 수치 일관성 이슈

| 항목 | 소스별 값 |
|-----|---------|
| Yuki 월 덕질 지출 | CREAT-yuki: 15% / 교정 PDF §수정2: 17,689엔 / 최종 프롬프트: 12% |

CREAT 문서 L99에 작성자 본인이 "통일 필요" 플래그를 남겼으나 미해결 상태로 최종 반영.
→ 향후 수정 시 17,689엔(실측 근거 있음) 기준으로 % 역산하여 통일.

### 1차 인터뷰 부재

`src/raw/interviews/` 디렉토리 비어있음.
모든 소스가 2차(정부 보고서, 딥리서치) 또는 1.5차(내부 패널 데이터, 내부 Q&A 템플릿).
실제 일본 현지 팬의 1차 인터뷰 음성/텍스트 없음.

→ 최우선 보완 과제: 현지 팬 인터뷰 1~3건 확보 → `src/raw/interviews/` 적재.

### 창작 항목 혼재

두 퍼소나 모두 리서치 근거 없는 창작 디테일 포함 (가상 그룹, 특정 기기 모델, 수치 임의 확정).
현재 문서에 표시 없음 → LLM이 이를 사실로 인식할 가능성.

→ 퍼소나 파일 내 창작 섹션에 주석 또는 별도 `[FICTIONAL]` 태그 추가 검토.

## 향후 퍼소나 작성 체크리스트

MINA, 또는 새 국가 퍼소나 추가 시 참조.

### Phase 1: 리서치 수집
- [ ] 정량 레이어: Weverse 패널 데이터 해당 국가/세그먼트 추출 (`insight-data-table.md` 갱신)
- [ ] 정성 레이어: 해당 시장 2차 리서치 3개 이상 (정부 보고서, 학술, 미디어)
- [ ] 1차 레이어: 현지 팬 인터뷰 최소 1건 (`src/raw/interviews/` 적재)

### Phase 2: 인사이트 추출
- [ ] 각 문서 핵심 인사이트 5개 추출 후 `02-research-to-insight.md` 형식으로 정리
- [ ] 복수 소스에서 교차 검증된 항목 vs 단일 소스 의존 항목 구분 표시
- [ ] 정량 데이터 → 퍼소나 정체성 연결 논리 명시 (Silent Watcher 변환 예시처럼)

### Phase 3: CREAT 질문 적용
- [ ] `CREAT-팬 퍼소나 제작 질문...md` 템플릿 적용
- [ ] Q4(앱 생태계) 답변은 반드시 독립 소스로 교차 검증
- [ ] 수치는 소스 명시 필수 (출처 없는 수치 기입 금지)

### Phase 4: 퍼소나 합성
- [ ] 트레이트별 소스 태그 첨부 (내부 작업 시트 유지)
- [ ] 창작 항목 `[FICTIONAL]` 명시 (가상 그룹명, 가상 멤버명, 특정 일화)
- [ ] 정량 미반영 항목은 이유 한 줄 명시 (예: "양극 페어링 목적")
- [ ] 수치 일관성 확인: 동일 지표가 복수 소스에서 다른 값이면 통일 후 기입

### Phase 5: 평가 실행 (현재 미실행)
- [ ] `scenarios/{persona}/` 에 시나리오 최소 3개 작성
- [ ] 퍼소나 × 시나리오 → 대화 실행 → `transcripts/` 적재
- [ ] `evaluation/judge.md` 로 R1~R6 + D1~D5 채점
- [ ] `evaluation/scores/` 결과 적재 및 약점 항목 퍼소나 개정
