# DS 토큰 사전 준비

`$ARGUMENTS`를 파싱해 플랫폼별 DS 토큰 파일을 생성한다.  
`web_ui_implementor` Step 0에서 자동 호출되거나, 하네스 실행 전 수동으로 실행한다.

## 실행 순서

### Step 1: 인자 파싱 + 플랫폼 결정

`$ARGUMENTS`에서 `--platform=<value>` 추출.  
없으면 현재 디렉터리 파일로 감지:
- `package.json` → `web`
- `Package.swift` 또는 `*.podspec` → `ios`
- `build.gradle` 또는 `build.gradle.kts` → `android`

### Step 2: 출력 경로 결정

- **web**: `src/tokens/wds-tokens.css` (기본값) 또는 `--output` 인자 경로
- **ios**: `Sources/Tokens/WDSTokens.swift` (기본값)
- **android**: `src/main/res/values/wds_tokens.xml` (기본값)

이미 존재하면: "이미 존재합니다. 덮어쓰려면 `--force` 인자를 추가하세요." 보고 후 종료.

### Step 3: cds-catalogs 토큰 fetch

`cds-ds` MCP 서버가 있으면 `ds_list()` 호출로 연결 확인.  
없으면 GitHub API 직접 사용:

```
GET https://api.github.com/repos/weversecorp/cds-catalogs/contents/catalogs/tokens
Authorization: Bearer <gh auth token 결과>
```

`.json` 확장자 파일 목록 수집. 각 파일을 순차 fetch:

```
GET https://api.github.com/repos/weversecorp/cds-catalogs/contents/catalogs/tokens/<file>.json
```

응답의 `content` 필드를 base64 디코딩해 JSON 파싱.  
구조: `{ "tokens": [{ "name": "system/color/...", "values": { "light": { "raw": "#HEX" }, "dark": { "raw": "#HEX" } } }] }`

GitHub 인증 실패 시:
```
⚠️  GitHub 인증이 필요합니다.
터미널에서 실행: gh auth login
이후 Claude Code를 재시작하거나 GITHUB_TOKEN 환경변수를 설정하세요.
```

### Step 4: 플랫폼별 토큰 파일 생성

#### Web — `src/tokens/wds-tokens.css`

변환 규칙: `name.replace('/', '-')` → `--wds-<kebab>` CSS 변수

```css
/* Generated by figma-impl/impl-tokens — DO NOT EDIT MANUALLY */
/* Source: weversecorp/cds-catalogs/catalogs/tokens */

:root {
  /* <collection-name> */
  --wds-system-color-button-default: #3D5AFE;
  --wds-system-color-surface-primary: #FFFFFF;
  /* ... */
}

[data-theme="dark"] {
  --wds-system-color-button-default: #6E8BFF;
  --wds-system-color-surface-primary: #121212;
  /* ... */
}
```

값이 없는 토큰(`raw`가 빈 문자열이거나 `null`): `/* no value */` 주석으로 생략.

`src/` 디렉터리가 없으면 `src/tokens/` 경로 전체를 생성.

#### iOS — `Sources/Tokens/WDSTokens.swift` (참고용, 구현 선택적)

```swift
// Generated by figma-impl/impl-tokens — DO NOT EDIT MANUALLY
import UIKit
public enum WDSTokens {
  public enum Color {
    public static let systemColorButtonDefault = UIColor(hex: "#3D5AFE")
    // ...
  }
}
```

#### AOS — `src/main/res/values/wds_tokens.xml` (참고용, 구현 선택적)

```xml
<!-- Generated by figma-impl/impl-tokens — DO NOT EDIT MANUALLY -->
<resources>
  <color name="wds_system_color_button_default">#3D5AFE</color>
</resources>
```

### Step 5: 진입점에 import 추가 (web 전용)

`src/main.tsx` 또는 `src/main.ts` 파일을 Read.  
`import './tokens/wds-tokens.css'` 라인이 없으면 첫 번째 import 블록 앞에 추가.

`src/App.tsx`에서도 import 없는 경우 동일하게 추가.

### Step 6: 완료 보고

```
## DS Token Setup 완료

- 플랫폼: web
- 토큰 파일 수: N개 (color.json, size.json, typography.json, ...)
- 생성된 CSS 변수: N개 (light), N개 (dark)
- 출력 파일: src/tokens/wds-tokens.css
- import 추가: src/main.tsx ✅

### 경고 (있을 경우)
- <token-name>: raw 값 없음 — 생략됨
- GitHub rate limit 근접 시 경고
```

## 오류 처리

| 상황 | 처리 |
|------|------|
| GitHub 인증 실패 | 인증 안내 메시지 출력 후 중단 |
| 토큰 파일 0개 | "catalogs/tokens 디렉터리가 비어있습니다" 경고 |
| JSON 파싱 실패 | 해당 파일 스킵 + 경고 기록 |
| `src/` 없음 | 디렉터리 자동 생성 |
| 이미 존재 + `--force` 없음 | 덮어쓰기 중단, 안내 메시지 |
