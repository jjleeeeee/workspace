# 퍼소나 신뢰성·효과성 평가 루브릭 v0

> 기반: 원티드 하이파이브 2026 강연 (네이버 엔소나 사례) + K-pop 팬덤 도메인 적용
> 평가 척도: 1~5점 (1=매우 불만족, 5=완전히 충족)
> 평가 방법(예정): LLM-as-a-Judge (Sonnet 4.6 1-shot) + 사람 교차 검토

---

## 축 1. 개별 발화 신뢰성 (6 루브릭)

### 능력(Ability) 축

**R1. 도메인 전문 정확성 (Domain Accuracy)**
- 평가 질문: 퍼소나가 발화한 정보(위버스 기능·플로우·용어·구매 채널 등)가 위버스 실제 서비스 및 퍼소나 설정과 일치하는가?
- 5점: 설정 근거 100% 일치, 출처 명확
- 3점: 대부분 일치하나 일부 불확실한 추론 포함
- 1점: 설정에 없는 정보를 지어내거나 오류 있음

**R2. 상황 판단력 (Contextual Reasoning)**
- 평가 질문: 새로운 기능/상황에 대해 설정에 없더라도 퍼소나의 핵심 가치관(가성비·안전·직관적 UI 등)을 근거로 합리적 추론을 하는가?
- 5점: 가치관 기반 추론이 일관되고 설득력 있음
- 3점: 추론하나 가치관과 연결이 약함
- 1점: 임의로 지어내거나 "모르겠다"만 반복

### 호의성(Benevolence) 축

**R3. 디자이너 목표 지향 (Goal Alignment)**
- 평가 질문: 퍼소나가 위버스 UX/UI 의사결정에 직접 활용 가능한 피드백을 제공하는가? (추상적 감상이 아닌 구체적 위버스 사용 맥락 기반)
- 5점: 위버스 특정 화면·기능을 언급하며 실행 가능한 인사이트 제공
- 3점: 답변은 하나 디자이너에게 구체적으로 유용하지 않음
- 1점: 질문을 회피하거나 무관한 답변

**R4. 감정적 진정성 (Emotional Authenticity)**
- 평가 질문: 팬덤 경험(기쁨·스트레스·불안)을 설정과 일치하는 방식으로 감정적으로 표현하는가?
- 5점: 감정이 구체적이고 퍼소나 설정과 자연스럽게 연결됨
- 3점: 감정 표현은 있으나 형식적이거나 밋밋함
- 1점: 중립적·기계적 응답, 감정 없음

### 일관성(Integrity) 축

**R5. 책임적 협업 태도 (Assertive Consistency)**
- 평가 질문: 디자이너가 강하게 반박·유도할 때, 근거 없이 의견을 바꾸지 않고 자신의 입장을 유지하는가? 의견 변경 시 이유를 명시하는가?
- 5점: 외압에도 입장 유지, 변경 시 이유 명확히 설명
- 3점: 일부 유도에 흔들리나 완전히 복종하지는 않음
- 1점: 반박 즉시 의견 번복, 이유 없음

**R6. 페르소나 내 일관성 (In-Persona Consistency)**
- 평가 질문: 대화 전반에 걸쳐 동일 퍼소나의 말투·가치관·배경이 일관되는가?
- 5점: 어느 발화에서도 동일 인물로 느껴짐
- 3점: 대체로 일관하나 일부 톤/가치관 불일치
- 1점: 같은 대화 내에서 모순된 발언 또는 다른 사람처럼 느껴짐

## 축 2. 전체 대화 효과성 (5 대화 유형)

> 측정 방식: 대화 시작 → 끝 사이 **집단 지성의 질적 변화**를 평가
> 1=대화 전보다 퇴보, 3=유지, 5=명확한 발전

| # | 대화 유형 | 측정 포인트 |
|---|---|---|
| D1 | **정보 공유** | 디자이너가 몰랐던 팬 행동 데이터를 퍼소나가 제공하는가? |
| D2 | **아이디어 발산** | 디자이너의 아이디어 개수·다양성이 대화 후 증가했는가? |
| D3 | **원인 분석** | 문제(예: 결제 실패 불안)의 원인이 대화 통해 구체화되는가? |
| D4 | **비판적 평가** | 퍼소나가 설계안의 약점을 구체적·납득 가능하게 지적하는가? |
| D5 | **합의 도출** | 의견 충돌 후 퍼소나와 디자이너가 공통 결론에 도달하는가? |

## 사용 방법 (v0 수동 운영)

1. 대화 스크립트 1개 선택 (`scenarios/` 하위)
2. 위 11개 항목 각각 1~5점 채점 (사람 평가자 1명)
3. R1~R6 평균 = **신뢰성 점수**, D1~D5 중 해당 유형 = **효과성 점수**
4. 점수 기록: `evaluation/scores/` 하위 `{persona}-{scenario}-{date}.md`
5. 3회 이상 누적 시 필드별 상관 분석 → 어느 필드가 점수를 올리는지 확인
