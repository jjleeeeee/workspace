## 2. harness-cli code-connect resolve — mainComponent fallback

### 문제
- `code-connect-index.json` 키: DS 파일 `DWEduE6GfxYMlyxKPNJ8jA`
- 샵 파일 인스턴스 nodeId (`VHF5kgkVRAtctETQfeVL1W` 기준) → index 미매칭
- 원인: 인스턴스 nodeId ≠ DS 파일 main component nodeId

### 해결 (`scripts/harness-cli.mjs`)

**1. 워커 스크립트** (`buildWorkerScript`):
```js
// dsInstancesSkipped 에 mainComponentId 추가
skipped.push({
  id: node.id, name: node.name,
  mainComponentId: node.mainComponent?.id ?? null,  // ← 신규
  mainComponentKey: node.mainComponent?.key ?? null,
  mainComponentName: node.mainComponent?.name ?? null
});
```

**2. `runCodeConnectResolve` fallback**:
- 직접 lookup 실패 시 → `harness/*-nodes.json` 전체 스캔
- `dsInstancesSkipped`에서 일치하는 instance id 탐색
- `mainComponentId`(= DS 파일 nodeId)로 index 재조회
- 성공 시 `_resolvedViaMainComponent` 메타 포함 반환

### 동작 조건
`extract-nodes` 배치 완료 후 `dsInstancesSkipped`에 DS 인스턴스 기록 → 이후 `code-connect resolve` 자동 동작.

### 발견 및 수정한 버그

| # | 위치 | 버그 | 수정 |
|---|------|------|------|
| 1 | GNB 뒤로가기 | `arrowDownFoldMedium` (∧) 사용 | `arrowLeftMedium` (←) 으로 교체 |
| 2 | 플로팅 버튼 | `favoritesMedium` (☆) 아이콘 | `arrowDownMedium` + `transform: rotate(180deg)` CSS |
| 3 | 탭바 누락 | 하단 탭바 없음 | `WeverseBottomNav` import 후 sticky footer 안에 추가 |

### 레이아웃 구조 변경
```
Before:
  footer.bottom (sticky)
    floatingBtnRow (scroll-to-top)
    bottomBar (결제 버튼)
  WeverseBottomNav (DOM 밖 — sticky footer에 가려짐)

After:
  div.body (scrollable)
    ...cards...
    floatingBtnRow (body 내 스크롤 영역 끝)
  footer.bottom (sticky bottom: 0)
    bottomBar (결제 버튼 + terms)
    WeverseBottomNav (탭바)
```
