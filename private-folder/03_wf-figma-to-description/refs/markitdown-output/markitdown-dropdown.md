# [Usage] Dropdown

Dropdown은 사용자가 여러 항목을 확인하고 그중 하나를 선택하기 위해 사용하는 선택 컴포넌트입니다.

## Context

Dropdown은 선택지가 화면에 항상 노출될 필요는 없지만, 사용자가 필요할 때 목록을 열어 선택해야 하는 상황에 사용해야 합니다. Text Field처럼 값을 입력하는 컴포넌트가 아니라, 제공된 항목 중 선택하게 하는 컴포넌트로 사용해야 합니다.

## Principle

- 선택 가능한 목록과 현재 선택값을 명확하게 구분해야 합니다.
- Dropdown은 기준이 되는 필드나 트리거와 연결된 위치에 노출해야 합니다.
- 항목의 정보는 사용자가 비교하고 선택할 수 있을 만큼 충분히 명확해야 합니다.

## Usage Do / Don't

### 선택값

Do

- 현재 선택된 값을 트리거 영역에 명확히 표시해야 합니다.

Don't

- 선택 후에도 Placeholder처럼 보이게 두지 않아야 합니다. (선택 완료 여부 혼동)

### 목록

Do

- 사용자가 선택해야 하는 항목을 Dropdown 목록 안에 일관된 형식으로 제공해야 합니다.

Don't

- 입력 설명, 안내문, 선택 항목을 같은 레벨로 섞지 않아야 합니다. (정보 구조 혼동)

### Position

Do

- Dropdown은 기준이 되는 입력 필드나 트리거와 연결된 위치에 노출해야 합니다.

Don't

- 트리거와 분리된 위치에 Dropdown을 띄우지 않아야 합니다. (관계 인지 어려움)

### Status

Do

- Dropdown은 연결된 필드의 상태와 컬러 규칙을 따라야 합니다.

Don't

- 상태와 무관한 컬러를 임의로 적용하지 않아야 합니다. (상태 의미 혼동)

### 선택 범위

Do

- 사용자가 제공된 항목 중 선택해야 할 때 Dropdown을 사용해야 합니다.

Don't

- 자유 입력이 필요한 값에 Dropdown만 제공하지 않아야 합니다. (입력 제약 오류)
