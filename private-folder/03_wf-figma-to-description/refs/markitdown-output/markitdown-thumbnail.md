# [Usage] Thumbnail

Thumbnail은 콘텐츠를 시각적으로 대표하는 이미지 요소를 일정한 비율과 크기로 보여주기 위해 사용하는 이미지 컴포넌트입니다.

## Context

Thumbnail은 콘텐츠 목록, 카드, 미디어, 상품처럼 사용자가 이미지를 보고 대상을 빠르게 식별해야 할 때 사용해야 합니다. 원본 이미지의 자유 비율을 그대로 보여주는 용도가 아니라, 정해진 Square 또는 Rectangular 비율 안에서 콘텐츠 대표 이미지를 안정적으로 보여주는 용도로 사용해야 합니다. Thumbnail 위에 아이콘, Seek Bar, Button Area 같은 선택 요소를 올릴 때는 모든 비율에 동일한 스펙을 적용해야 합니다.

## Principle

- 이미지 비율에 따라 Square와 Rectangular를 구분하고 1:1, 3:4, 5:6, 5:8, 9:16, 16:9 중 선택해야 합니다.
- 모든 이미지 비율은 가로 길이 256 기준으로 관리하고, 최소 길이는 가로 44 기준을 지켜야 합니다.
- 소수점이 나올 경우 어떠한 경우에도 반내림해야 합니다.
- 선택 요소는 모든 썸네일 비율에 동일한 스펙으로 적용하고, 아이콘은 최대 2개까지만 노출해야 합니다.

## Usage Do / Don't

### Ratio

Do

- 콘텐츠 성격에 맞춰 1:1, 3:4, 5:6, 5:8, 9:16, 16:9 중 적절한 비율을 선택해야 합니다.

Don't

- 같은 목록 안에서 Square와 Rectangular를 임의로 섞지 않아야 합니다. (레이아웃 불안정)

### Size & Rounding

Do

- 모든 이미지의 비율은 가로 길이 256 기준으로 맞추고, 소수점이 나올 경우 반내림해야 합니다.

Don't

- 원본 이미지 크기에 따라 Thumbnail 비율을 매번 바꾸거나 소수점을 반올림하지 않아야 합니다. (시각 일관성 저하)

### Minimum

Do

- Thumbnail의 최소 길이는 가로 44 기준을 지켜야 합니다.

Don't

- 가로 44보다 작게 줄이지 않아야 합니다. (식별성 저하)

### Placeholder

Do

- 이미지가 없는 상태나 로딩 상태는 지정된 상태 컬러와 대체 표현을 사용해야 합니다.

Don't

- 깨진 이미지나 빈 영역을 그대로 노출하지 않아야 합니다. (완성도 저하)

### Crop

Do

- 대표 대상이 잘 보이도록 정해진 비율 안에서 이미지를 배치해야 합니다.

Don't

- 핵심 피사체가 잘리도록 무리하게 크롭하지 않아야 합니다. (콘텐츠 식별 실패)

### Optional Items

Do

- Thumbnail 위 선택 요소는 모든 비율에 동일한 스펙으로 적용하고, 아이콘은 최대 2개까지만 노출해야 합니다.

Don't

- 아이콘을 3개 이상 노출하거나 선택 요소 위치를 임의로 바꾸지 않아야 합니다. (콘텐츠 가림 및 조작 혼동)
