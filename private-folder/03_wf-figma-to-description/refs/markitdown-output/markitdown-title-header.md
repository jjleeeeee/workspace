# [Usage] Title Header

Title Header는 화면 안에서 제목과 정보를 표시할 수 있는 공간을 제공하기 위해 사용하는 패턴입니다.

## Context

Title Header는 페이지 이동을 담당하는 Top Navigation과 달리, 현재 화면 안에서 콘텐츠 묶음의 제목이나 정보를 보여줄 때 사용해야 합니다. 다른 페이지로 이동할 수 없는 단순 헤더 역할만 하며, Left 또는 Center 타입 중 화면 내 정보 위계에 맞는 정렬을 선택해야 합니다. 복잡한 탐색 구조나 여러 줄 설명을 담기 위한 영역으로 사용하지 않아야 합니다.

## Principle

- 높이는 48px로 고정해 주변 레이아웃을 흔들지 않아야 합니다.
- 다른 페이지로 이동할 수 없는 단순 헤더 역할만 해야 합니다.
- 현재 화면의 제목과 정보만 노출해야 합니다.
- 화면에 고정되는 영역으로 사용하지 않아야 합니다.

## Usage Do / Don't

### Type

Do

- 화면 내 정보 위계에 맞춰 Left 또는 Center 타입을 선택해야 합니다.

Don't

- 같은 화면에서 Left와 Center 정렬을 임의로 섞지 않아야 합니다. (제목 위계 혼동)

### 제목 길이

Do

- 짧은 텍스트 타이틀은 최대 1줄로 사용해야 합니다.

Don't

- 2줄 이상의 텍스트를 Title Header에 넣지 않아야 합니다. (높이 고정 규칙 훼손)

### 역할 범위

Do

- 현재 화면의 제목과 정보만 노출하고 다른 페이지 이동 역할은 배제해야 합니다.

Don't

- 페이지 이동, 복잡한 탐색, 본문성 문장을 Title Header에 넣지 않아야 합니다. (헤더 역할 혼동)

### 액션

Do

- 보조 액션이 필요한 경우 텍스트 버튼이나 Drop Down을 함께 사용해야 합니다.

Don't

- 헤더 역할을 벗어나는 복잡한 액션을 여러 개 배치하지 않아야 합니다. (조작 부담 증가)

### 고정 영역

Do

- Title Header는 콘텐츠 흐름 안에서 제목과 정보를 표시하는 영역으로 사용해야 합니다.

Don't

- 화면 상단에 고정되는 내비게이션 영역처럼 사용하지 않아야 합니다. (Top Navigation 역할 중복)

### Avatar 조합

Do

- 아바타와 텍스트를 함께 사용할 때는 대상 식별을 돕는 맥락에서만 사용해야 합니다.

Don't

- 장식 목적으로 아바타를 추가하지 않아야 합니다. (식별 의미 약화)
