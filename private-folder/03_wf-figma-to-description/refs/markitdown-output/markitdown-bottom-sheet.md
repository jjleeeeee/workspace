# [Usage] Bottom Sheet

Bottom Sheet는 화면 아래에서 올라와 이전 화면의 맥락을 유지하면서 추가 정보, 선택, 하위 흐름을 제공하기 위해 사용하는 오버레이 컴포넌트입니다.

## Context

Bottom Sheet는 Bottom Popup보다 더 많은 정보나 내부 Depth가 필요한 상황에 사용해야 합니다. 사용자가 화면 높이를 조정하거나 내부에서 다음 단계로 이동해야 하는 흐름이라면 Bottom Sheet가 적합합니다.

## Principle

- Bottom Sheet 안에서는 Depth 구성이 가능해야 합니다.
- 정보량에 따라 사용자가 화면 높이를 최대로 조정할 수 있어야 합니다.
- 사용자가 원할 때 닫기 아이콘과 제스처로 쉽게 닫을 수 있어야 합니다.

## Usage Do / Don't

### Depth

Do

- 하위 단계나 복수 화면 흐름이 필요한 경우 Bottom Sheet를 사용해야 합니다.

Don't

- 단순 안내만 필요한 상황에 Bottom Sheet를 사용하지 않아야 합니다. (컴포넌트 과사용)

### 하단 마진

Do

- 콘텐츠 영역 하단 마진값은 24px로 고정해야 합니다.

Don't

- 콘텐츠 영역 하단 마진값을 변경하지 않아야 합니다. (레이아웃 일관성 저하)

### 타이틀 위치

Do

- iOS 타이틀은 중앙, AOS 타이틀은 좌측에 고정해야 합니다.

Don't

- OS별 타이틀 위치 규칙을 섞어 사용하지 않아야 합니다. (플랫폼 일관성 저하)

### 닫기 아이콘

Do

- 닫기 아이콘을 포함해 다양한 제스처로 닫을 수 있게 해야 합니다.

Don't

- 닫기 아이콘을 제거하지 않아야 합니다. (닫기 가능성 인지 저하)

### 위치

Do

- Bottom Sheet는 기본적으로 화면 하단에 위치하고 좌우 최대 너비는 410px이어야 합니다.

Don't

- Bottom Sheet를 화면 중앙 팝업처럼 배치하지 않아야 합니다. (컴포넌트 의미 오용)
