# [Usage] Tag

Tag는 콘텐츠의 속성, 카테고리, 상태 등을 짧은 키워드로 식별하거나 분류하기 위해 사용하는 정보 표시 컴포넌트입니다.

## Context

Tag는 사용자가 콘텐츠의 성격이나 상태를 빠르게 스캔해야 할 때 사용해야 합니다. Small과 Medium 사이즈 중 정보 위계와 노출 위치에 맞는 사이즈를 선택해야 하며, 동일 맥락 안에서는 형태와 사이즈를 통일해야 합니다. 선택 가능한 옵션처럼 보이면 Chips를 사용해야 하며, 상태보다 알림이나 카운트가 중요하면 Badge를 사용해야 합니다.

## Principle

- 태그 텍스트는 핵심 키워드 위주로 짧고 명확해야 합니다.
- 정의된 시스템 컬러 토큰만 사용해야 합니다.
- 불투명한 배경 색상을 사용할 경우 텍스트 가시성을 고려해야 합니다.
- 동일 맥락 안에서는 태그 형태와 사이즈를 통일해야 합니다.

## Usage Do / Don't

### Text

Do

- 태그 내 텍스트는 핵심 키워드 위주로 짧게 작성해야 합니다.

Don't

- 문장형 설명을 Tag 안에 넣지 않아야 합니다. (스캔성 저하)

### Size

Do

- Small 또는 Medium 중 맥락에 맞는 정해진 사이즈를 사용해야 합니다.

Don't

- 동일 맥락 안에서 Small과 Medium을 임의로 섞지 않아야 합니다. (정보 위계 혼동)

### Color Token

Do

- 정의된 system/fixed_color/roles 컬러 토큰을 사용하고 불투명 배경에서는 텍스트 가시성을 확인해야 합니다.

Don't

- 임의 색상이나 대비가 약한 배경색을 적용하지 않아야 합니다. (텍스트 가독성 저하)

### Radius

Do

- 각 Shape의 지정된 Radius 값을 사용해야 합니다.

Don't

- Radius 값을 임의로 조정하지 않아야 합니다. (형태 일관성 저하)

### Icon Size

Do

- 아이콘이 포함될 경우 지정된 아이콘 크기(tiny10)를 사용해야 합니다.

Don't

- Tag 안의 아이콘 크기를 임의로 키우지 않아야 합니다. (텍스트 위계 침범)

### Gray Color

Do

- Gray 컬러는 system/fixed_color/text/gray-500으로 제한해야 합니다.

Don't

- Text와 Outline에 임의의 Gray 컬러 값을 넣지 않아야 합니다. (dirty gray 방지)
