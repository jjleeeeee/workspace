# [Usage] In App Browser

In App Browser는 앱 안에서 일시적으로 웹 페이지를 보여주기 위해 사용하는 브라우저 화면 컴포넌트입니다.

## Context

In App Browser는 사용자가 앱을 완전히 이탈하지 않고 외부 웹 콘텐츠를 확인해야 할 때 사용해야 합니다. 외부 웹의 Back/Close 기능과 앱 내부 네비게이션이 중복되지 않도록 OS별 헤더 규칙을 따라야 합니다.

## Principle

- 앱에 종속된 브라우저임을 이해할 수 있도록 헤더와 닫기 기능을 명확히 제공해야 합니다.
- OS별 닫기 버튼 위치와 텍스트 정렬 규칙을 따라야 합니다.
- 헤더는 스크롤과 관계없이 화면 상단에 고정되어야 합니다.

## Usage Do / Don't

### Close Button

Do

- Android는 개발 특성에 맞춰 좌측 닫기 버튼만 사용해야 합니다.

Don't

- Android에서 우측에 닫기 버튼을 배치하지 않아야 합니다. (플랫폼 규칙 위반)

### Text Alignment

Do

- Android의 텍스트는 좌측 정렬해야 합니다.

Don't

- Android에서 텍스트를 중앙 정렬하지 않아야 합니다. (플랫폼 일관성 저하)

### 기능 중복

Do

- 기획팀과 논의해 Back 또는 Close 버튼을 생략할 수 있어야 합니다.

Don't

- 동일한 기능을 중복해서 노출하지 않아야 합니다. (Back 이중 노출)

### Header

Do

- 헤더는 스크롤과 관계없이 화면 상단에 고정해야 합니다.

Don't

- 헤더를 스크롤 영역 안에 포함하지 않아야 합니다. (닫기 기능 접근성 저하)

### 사용 범위

Do

- 앱 안에서 일시적으로 웹 페이지를 보여줘야 할 때 사용해야 합니다.

Don't

- 앱의 네이티브 주요 화면을 In App Browser로 대체하지 않아야 합니다. (탐색 구조 혼동)
