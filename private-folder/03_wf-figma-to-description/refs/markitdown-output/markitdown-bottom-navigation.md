# [Usage] Bottom Navigation

Bottom Navigation은 사용자가 앱의 주요 영역을 빠르게 오가기 위해 화면 하단에 고정해 사용하는 탐색 컴포넌트입니다.

## Context

Bottom Navigation은 서비스의 최상위 목적지가 3~5개처럼 반복적으로 접근되어야 할 때 사용해야 합니다. 현재 위치를 계속 인지해야 하는 앱 구조에 적합하며, 일회성 액션이나 페이지 안의 보조 이동에는 사용하지 않아야 합니다. Shop처럼 서비스 영역의 디스플레이 모드가 별도로 고정되는 경우에는 해당 서비스 조건을 우선 적용해야 합니다.

## Principle

- 주요 목적지를 빠르게 전환하기 위해 항상 같은 위치와 같은 순서로 노출해야 합니다.
- 사용자가 현재 위치를 인지하기 위해 활성 탭과 비활성 탭의 시각 차이가 명확해야 합니다.
- 탐색 의미를 이해하기 위해 아이콘과 레이블 텍스트를 함께 사용해야 합니다.
- 해상도에 대응하기 위해 각 탭 영역은 좌우로 유연하게 늘어나야 합니다.

## Usage Do / Don't

### 탭 개수

Do

- 핵심 탐색 목적지만 남기기 위해 고정된 탭 개수를 사용해야 합니다.

Don't

- 임시 메뉴나 캠페인 진입점을 탭으로 추가하지 않아야 합니다. (탐색 구조 혼동)

### 구성 및 레이블

Do

- 각 탐색 옵션은 아이콘과 지정된 너비 안의 짧은 레이블을 함께 제공해야 합니다.

Don't

- 레이블 없이 아이콘만 노출하거나 긴 레이블을 줄바꿈/말줄임 처리하지 않아야 합니다. (의미 추측 및 가독성 저하)

### 활성 상태

Do

- 현재 위치를 알리기 위해 활성 탭은 Filled 아이콘과 Bold 레이블로 표시해야 합니다.

Don't

- 활성/비활성 탭에 같은 아이콘 스타일과 굵기를 사용하지 않아야 합니다. (현재 위치 식별 어려움)

### 디스플레이 모드

Do

- Shop은 라이트 모드로 고정되어야 하며, 다크 모드에서 Shop으로 진입해도 라이트 모드로 노출되어야 합니다.

Don't

- Shop Bottom Navigation을 앱 다크 모드에 맞춰 다크 컬러로 전환하지 않아야 합니다. (서비스 테마 일관성 저하)

### 좌우 Flexible

Do

- Bottom Navigation은 해상도에 맞게 각 탭 영역의 좌우가 늘어나는 구조로 사용해야 합니다.

Don't

- 특정 해상도 기준의 고정폭으로 탭 영역을 배치하지 않아야 합니다. (반응형 대응 부족)

### Badge

Do

- 알림이나 업데이트를 표시하기 위해 Badge는 정해진 위치와 스타일로만 사용해야 합니다.

Don't

- Badge 위치나 컬러를 임의로 변경하지 않아야 합니다. (상태 의미 혼동)
