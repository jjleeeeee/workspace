# [Usage] Bottom Popup

Bottom Popup은 화면 아래에서 올라와 현재 화면을 벗어나지 않고 간단한 정보나 선택지를 제공하기 위해 사용하는 오버레이 컴포넌트입니다.

## Context

Bottom Popup은 추가 정보나 짧은 선택을 제공하지만, 내부에서 새로운 Depth를 구성하지 않아야 하는 상황에 사용해야 합니다. 내부에서 단계 이동이나 복잡한 흐름이 필요하면 Bottom Sheet를 사용해야 합니다.

## Principle

- 사용자가 현재 화면의 맥락을 유지한 채 추가 정보를 확인할 수 있어야 합니다.
- Bottom Popup 내부에서는 Depth 구성을 하지 않아야 합니다.
- 사용자가 원할 때 쉽게 닫을 수 있어야 합니다.

## Usage Do / Don't

### Depth

Do

- 한 화면 안에서 끝나는 간단한 정보나 선택에 사용해야 합니다.

Don't

- Popup 안에서 다음 화면으로 이동하는 Depth를 구성하지 않아야 합니다. (흐름 구조 혼동)

### 하단 마진

Do

- 콘텐츠 영역 하단 마진값은 24px로 고정해야 합니다.

Don't

- 콘텐츠 영역 하단 마진값을 변경하지 않아야 합니다. (레이아웃 일관성 저하)

### 타이틀

Do

- 타이틀은 좌측에 고정해야 합니다.

Don't

- 타이틀 위치를 중앙이나 우측으로 변경하지 않아야 합니다. (OS/패턴 일관성 저하)

### 닫기 방식

Do

- 사용자가 다양한 제스처로 닫을 수 있게 해야 합니다.

Don't

- 별도 닫기 아이콘을 추가하지 않아야 합니다. (컴포넌트 규칙 위반)

### 위치

Do

- Android는 하단에서 20px 떨어지고, Bottom Popup의 좌우 최대 너비는 410px이어야 합니다.

Don't

- Bottom Popup을 화면 중앙이나 상단에 배치하지 않아야 합니다. (컴포넌트 의미 오용)
