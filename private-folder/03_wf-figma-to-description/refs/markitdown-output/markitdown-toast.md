# [Usage] Toast

Toast는 사용자가 수행한 작업에 대한 피드백을 제공하고, 일정 시간이 지나면 자동으로 사라지게 하기 위해 사용하는 피드백 컴포넌트입니다.

## Context

Toast는 오류, 완료, 진행상태처럼 사용자의 흐름을 막지 않아야 하는 짧은 시스템 상태 알림에 사용해야 합니다. 사용자가 후속 액션을 선택해야 하면 Snackbar를 사용하고, 결정을 내려야 하면 Dialog를 사용해야 합니다. 텍스트 길이에 따라 너비가 조절되며, 화면 좌우 마진과 해상도별 최대 너비를 지켜야 합니다.

## Principle

- 오류, 완료, 진행상태 메시지와 같은 시스템 상태 알림 텍스트를 제공해야 합니다.
- 간결한 문장으로 메시지를 전달하여 사용자 흐름을 방해하지 않아야 합니다.
- 일정 시간이 지나면 자동으로 사라져야 합니다.
- 기본 Usage에서는 아이콘이나 버튼 없이 텍스트 중심으로 정보를 전달해야 합니다.

## Usage Do / Don't

### 노출 위치

Do

- Toast는 화면 하단에 노출하고 하단 UI와 20px 간격을 유지한 뒤 일정 시간이 지나면 사라져야 합니다.

Don't

- Toast를 팝업처럼 사용자의 행동을 막도록 사용하지 않아야 합니다. (사용 흐름 차단)

### Status Message

Do

- 오류, 완료, 진행상태처럼 즉시 알려야 하는 시스템 상태 메시지를 간결하게 전달해야 합니다.

Don't

- 사용자의 선택이나 후속 조치가 필요한 내용을 Toast로 전달하지 않아야 합니다. (Snackbar/Dialog 역할 혼동)

### Text

Do

- Text는 최대 2줄까지 노출해야 합니다.

Don't

- 긴 설명이나 여러 문장을 Toast에 넣지 않아야 합니다. (가독성 저하)

### Action

Do

- Toast는 텍스트로 간결하게 정보를 전달해야 합니다.

Don't

- Toast 안에 아이콘이나 버튼을 넣지 않아야 합니다. (Snackbar 역할 혼동)

### Width

Do

- 텍스트가 길어져 최대 너비에 도달할 경우 자동 줄바꿈하고, 모바일에서는 좌우 10px 마진을 남겨야 합니다.
- 태블릿이나 PC 해상도에서는 Toast의 최대 너비를 410px로 제한해야 합니다.

Don't

- 최대 너비에 도달하지 않은 상태에서 강제 줄바꿈하거나 최대 너비를 초과하지 않아야 합니다. (읽기 흐름 저하)
- 큰 해상도에서 Toast를 화면 너비만큼 늘리지 않아야 합니다. (메시지 집중도 저하)

### Alignment

Do

- Toast 텍스트는 중앙 정렬로 입력해야 합니다.

Don't

- Toast 텍스트를 좌측이나 우측 정렬로 입력하지 않아야 합니다. (패턴 일관성 저하)
