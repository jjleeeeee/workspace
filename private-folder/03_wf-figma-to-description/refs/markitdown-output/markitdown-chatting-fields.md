# [Usage] Chatting Fields

Chatting Fields는 채팅, 댓글, 메시지 입력처럼 대화 맥락에서 텍스트를 입력하고 전송하기 위해 사용하는 입력 컴포넌트입니다.

## Context

Chatting Fields는 일반 Text Fields보다 즉시 전송, 보조 아이콘, 키보드 상태, 입력 가능 여부가 더 중요할 때 사용해야 합니다. 사용자가 지금 메시지를 작성 중인지, 전송할 수 있는지, 보조 액션을 사용할 수 있는지 명확히 알 수 있어야 합니다. 전송 버튼은 커뮤니티별 장식 컬러가 아니라 시스템 버튼 컬러 규칙을 따라야 합니다.

## Principle

- 입력과 전송 흐름을 방해하지 않기 위해 핵심 액션은 텍스트 필드 주변에 명확하게 배치해야 합니다.
- 전송 가능 상태를 알리기 위해 버튼 상태와 컬러를 일관되게 사용해야 합니다.
- 보조 아이콘은 메시지 작성 맥락과 직접 관련될 때만 제공해야 합니다.

## Usage Do / Don't

### 입력 목적

Do

- 댓글, 채팅, 메시지처럼 대화형 텍스트 입력에 사용해야 합니다.

Don't

- 일반 폼 입력이나 검색 입력에 Chatting Fields를 사용하지 않아야 합니다. (입력 목적 혼동)

### 전송 버튼 컬러

Do

- 전송 버튼의 컬러는 `system/color/button/default`로 사용해야 합니다.

Don't

- 전송 버튼을 커뮤니티 색상으로 변경하지 않아야 합니다. (서비스 액션 컬러 일관성 저하)

### Right Icon

Do

- 지정된 아이콘을 Text Field와 Right Icon 영역에 맞게 삽입해야 합니다.

Don't

- 전송 버튼과 같은 위치에 무관한 Right Icon을 동시에 노출하지 않아야 합니다. (액션 충돌)

### Left Icon

Do

- Left Icon은 첨부, 이모지처럼 메시지 작성에 필요한 보조 액션에만 사용해야 합니다.

Don't

- 입력과 무관한 메뉴 진입을 Left Icon으로 넣지 않아야 합니다. (조작 흐름 분산)

### 입력 상태

Do

- 입력 가능, 입력 중, 전송 가능 상태를 시각적으로 구분해야 합니다.

Don't

- 비어 있는 상태와 전송 가능한 상태를 같은 스타일로 표시하지 않아야 합니다. (전송 가능 여부 혼동)
