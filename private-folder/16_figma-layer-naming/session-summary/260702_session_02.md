## 해결한 것

### 1. Phase B 배치 스캔 도입
기존 `results.slice(0, 80)` 하드컷 → offset 배치 방식으로 교체.

**동작 방식**:
- `OFFSET=0, LIMIT=80`으로 첫 호출
- `hasMore: true`이면 `OFFSET += 80` 재호출 반복
- `hasMore: false`에서 종료, 누적 violations 전체로 Phase C 진행

**리턴 구조 변경**:
```js
// 기존
return { totalViolations: results.length, violations: results.slice(0, 80) };

// 변경 후
return {
  totalViolations: results.length,
  instancesSkipped: skipped.skipped,
  offset: OFFSET,
  hasMore: results.length > OFFSET + LIMIT,
  violations: results.slice(OFFSET, OFFSET + LIMIT)
};
```

### 2. Ponytail-review 반영
검토 4개 중 2개 채택:

| 항목 | 조치 |
|---|---|
| Phase C 예시 표에 BEM(`card__header`) 혼재 | `bannerSection`(camelCase)으로 교체, 사유 컬럼도 수정 |
| 플랫폼별 변환 참고 표 중복 | 삭제 (-5줄) |
| `skipped` 객체 구조 | 유지 (참조 전달 명확성) |
| OFFSET/LIMIT 주석 | 유지 (위치상 도움됨) |
