# 260702 Session 01 — Figma Layer Rename 워크플로우 구축

## 목표
AI 코드 생성 툴(Builder.io, Locofy 등) 대응을 위한 Figma 레이어 네이밍 표준화.  
선택된 프레임을 스캔 → dry-run → 승인 → 일괄 리네임하는 반복 워크플로우.

---

## 확정된 네이밍 규칙

| 카테고리 | 규칙 | 예시 |
|---|---|---|
| 최상위 화면 프레임 | PascalCase + 화면명+테마 | `FeedScreen`, `CommunityScreenLight` |
| 하위 레이아웃/요소 | camelCase (플랫폼 중립) | `tabSelectedBar`, `bannerSection` |
| Main Component | PascalCase | `CTAButton`, `UserAvatar` |
| INSTANCE | 스킵 (이름+자식 모두) | — |
| DS 노드 (💠/🛠️) | 스킵 (서브트리 전체) | — |

**BEM 불채택 이유**: CSS 전용. React Native / SwiftUI / Compose 환경에서 의미없음 → camelCase 통일.

**테마 전용명 블랙리스트**: `Light`, `Dark`, `Default`, `Dim`, `Auto` 단독 사용 금지 → `CommunityScreenLight` 처럼 화면명 포함.

---

## 스캔 로직 핵심

```javascript
// 보호 대상 (스킵)
if (isDSNode(node) || isDSInstance(node) || node.type === 'INSTANCE') → 스킵

// 판정 순서
1. auto-name: Frame/Group/Rectangle 등 + 숫자
2. has-space: 공백 포함
3. has-underscore: _ 포함
4. non-ascii: 한글 등 비ASCII
5. reserved-word: type/class/id/style 등
6. 최상위 프레임 or COMPONENT → PascalCase 강제, 테마명 단독 금지
7. 나머지 → camelCase 강제
```

**핵심 버그 수정 이력**:
- 1차: PascalCase 허용을 전 노드에 적용 → 하위 프레임(`Tabs`, `Union` 등) 누락
- 수정: PascalCase는 isTopLevelFrame || type==='COMPONENT' 만 허용
- 2차: INSTANCE 내부 재귀로 3920개 폭발 → INSTANCE 통째 스킵으로 45→24개
- 3차: 80개 상한 slice로 `파티 화력 슬롯` 등 누락 (미해결, 수동 대응)

## 워크플로우 (5 Phase)

1. **Phase A** — `figma_get_selection` 으로 선택 확인
2. **Phase B** — `figma_execute` JS로 서브트리 재귀 스캔, 위반 노드 수집
3. **Phase C** — dry-run 표 제시 (before → after), 승인 대기
4. **Phase D** — `figma_rename_node` 일괄 적용
5. **Phase E** — 재스캔으로 위반 0 확인

트리거: "리네임 시작" → 스킬 `figma-rename` 자동 실행  
스킬 경로: `~/.claude/skills/figma-rename/SKILL.md`
