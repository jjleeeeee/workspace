자료를 모으고, 질문으로 퍼소나를 채웁니다

자료를 수집하고, 변환하고, 인사이트를 뽑아 인터뷰 형식으로 정리합니다.
각 단계의 결과물이 다음 단계의 재료가 됩니다.

Stage 1

자료 수집

  • PDF, CSV, 노트 확보
  • 파일명·내용 수정 금지
  • 원본 = 읽기 전용

Stage 2

텍스트 변환

  • PDF·이미지 → 마크다운
  • 표·한자 누락 시 수동 복원
  • 변환본부터 편집 시작

Stage 3

인사이트 추출

  • 행동 패턴
  • 심리·동기
  • 불편한 점
  • 자주 쓰는 앱·채널

Stage 4

Q&A 작성

  • 기본 프로필 / 굿즈·소비
  • 콘텐츠 소비 / 앱 사용
  • 불편한 점 / 오프라인 활동

두 질문으로 나눠 점검합니다.

11개 항목별 채점 기준

11개 항목 각각 1~5점 채점. 5=완전 충족 / 3=부분 충족 / 1=미충족. LLM-as-a-Judge + 사람 교차 검토.

신뢰성 평가 (R1~R6)

도메인 정확성

위버스 기능·용어·구매 채널이 설정과 일치.
출처 근거 명확.

상황 판단력

설정 외 상황에서도 가치관(가성비·안전·직관적 UI)
기반으로 일관된 추론.

목표 지향

특정 화면·기능 언급하며 실행 가능한 인사이트.
추상적 감상 아닌 위버스 맥락 기반.

감정 진정성

기쁨·스트레스·불안을 설정과 연결해 구체적으로 표현.
중립적·기계적 응답 없음.

책임적 협업

강한 반박·유도에도 근거 없으면 입장 유지.
의견 변경 시 이유 한 문장 명시.

내적 일관성

대화 전반 동일 인물의 말투·가치관·배경 유지.
같은 대화 내 모순 발언 없음.

효과성 평가 (D1~D5)

정보 공유

디자이너가 몰랐던 팬 행동 데이터를 퍼소나가 제공.
대화 후 새로운 사실 파악 여부.

아이디어 발산

대화 후 디자이너 아이디어 개수·다양성 증가.
퍼소나 반응이 발상 확장에 기여.

원인 분석

문제(예: 결제 불안)의 원인이 대화로 구체화.
대화 전보다 원인 명확성 향상.

비판적 평가

설계안의 약점을 구체적·납득 가능하게 지적.
막연한 불만 아닌 근거 있는 피드백.

합의 도출

의견 충돌 후 퍼소나·디자이너 공통 결론 도달.
단순 동의 아닌 이유 있는 수렴.
