# 회의록

## 1. 핵심 요약

- Claude 라이선스 지급 지연으로 지난주 작업 진행에 제약이 있었다.
- CDS 카탈로그, DSL 스키마, MCP 서버, 컴포넌트 MD 파일 작업은 병렬로 진행 중이다.
- DSL 검증은 우선 웹을 기준 플랫폼으로 삼고, 이후 모바일 코드 생성으로 확장하는 방향을 검토한다.
- MD 파일은 계층 구조로 관리하고, 카탈로그 정보를 참조하는 방식이 현재 가장 유력하다.
- 다음 수요일 회의 전까지 결정 사항과 산출물을 정리하기로 했다.

---

## 2. 주요 이슈

### Claude 라이선스

- 대부분의 팀원이 아직 라이선스를 받지 못했다.
- 현재 협의 팀장 승인 단계에서 대기 중이다.
- 개인 환경에서 Cursor 등 무료 도구를 사용하는 것은 보안 정책 위반 가능성이 있어 금지로 확인했다.

### CDS 카탈로그

- 정주가 디자인 시스템 컴포넌트 카탈로그 전체 세트 구축을 완료했다.
- 배지, 아바타, 체크박스, 버튼, 라디오 버튼 등을 평가했다.
- 일부 컴포넌트에서 컬러와 투명도 이슈가 발견되었다.
- 우선 더미 카탈로그를 만든 뒤 지속 개선하기로 했다.

### DSL / MCP

- 지훈이 피그마 링크를 DSL로 변환하는 기능을 추가했다.
- 디자인 시스템 모듈의 매니페스트 생성/업데이트 명령을 구현했다.
- MCP 서버는 로컬 실행 방식으로 구축했고, `uv`를 사용한다.
- Penpot 구조는 참고하되, 자체 환경에 맞는 DSL 구조로 발전시킬 필요가 있다.

### MD 파일 구조

- 정주와 지영이 각자 컴포넌트 MD 파일을 제작 후 비교 중이다.
- 아바타, 배지, 버튼, 체크박스, 라디오 등 기본 컴포넌트를 먼저 작성했다.
- 이후 리스트 아이템, 타이틀 헤더 등 복합 컴포넌트로 확장 테스트 중이다.
- 최종 방향은 `MD 파일 계층 구조 + 카탈로그 참조` 방식이 유력하다.

### 검증 파이프라인

- 컴포넌트 하네스 합격 기준을 수립했다.
- 디자인 트랙과 컨트랙트 트랙으로 검증한다.
- Variant 축, 속성값 소스 검증을 포함해 전체 스코어 95점 이상을 통과 기준으로 본다.
- Playwright, Pixelmatch, PIL, 히트맵 등을 활용한다.
- 정확한 비교를 위해 화면 크기 일치가 필수다.

### 플랫폼별 현황

- Android: Gemini 2.5 Pro 적용 후 코드 생성 품질이 40~50% 수준으로 향상됐다. 스크롤 리스트 여백 이슈는 글로벌 규칙 추가가 필요하다.
- Web: DSL을 React 코드로 생성하는 스킬 개발을 검토한다.
- iOS: iOS 코드 생성 및 스킬 정리를 진행한다. JSON 카탈로그는 가독성 문제가 있어 MD 방식 선호 의견이 있었다.

## 3. 결정 사항

1. Claude 라이선스 지연으로 첫 스프린트는 3주로 연장한다.
2. 개인 환경의 무료 AI 개발 도구 사용은 보안상 금지한다.
3. DSL 검증은 웹을 기준 플랫폼으로 우선 진행한다.
4. MD 파일 계층 구조와 카탈로그 참조 방식을 우선 검토한다.
5. 피그마 MCP 단독 방식과 자체 개발 도구 방식은 AB 테스트한다.
