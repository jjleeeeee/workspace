# 퍼소나 제작 파이프라인 전체 구조

## 6단계 파이프라인

```
[A] 원본 자료 수집
    src/_origin/  (PDF, CSV 원본 — 불변 보존)
        ↓
[B] 마크다운 리서치 레이어
    src/*.md  (markitdown 변환 + 수동 복원)
        ↓
[C] CREAT 질문 템플릿 적용
    src/CREAT-팬 퍼소나 제작 질문 (for. WEJ, HBJ)...md
        ↓
[D] 퍼소나별 답변 시트
    src/CREAT-yuki-20s-full.md
    src/CREAT-satomi-40s-full.md
        ↓
[E] 시스템 인스트럭션 합성
    personas-by-country/japanese-yuki-20s.md
    personas-by-country/japanese-satomi-40s.md
        ↓
[F] 평가 레이어 (scaffolded — 미실행)
    evaluation/rubric-v0.md  →  evaluation/judge.md  →  evaluation/scores/
```

### [B] 마크다운 리서치 레이어

PDF → `markitdown` 자동 변환 후 수동 복원.
이미지 기반 PDF(도쿄25-09호)는 표 전체를 PDF 페이지 시각 읽기로 전사.
결과물: `src/` 내 8개 `.md` 파일.

### [C] CREAT 질문 템플릿

`CREAT-팬 퍼소나 제작 질문...md` — 6개 영역 bilingual Q&A 템플릿.

| Q번호 | 영역 |
|-------|------|
| Q1 | 기본 프로필 (나이, 거주지, 직업, 기기) |
| Q2 | 굿즈 소비 패턴 (선호 카테고리, 구매 동기) |
| Q3 | 콘텐츠 소비 (플랫폼, 시청 행태) |
| Q4 | 앱 생태계 (사용 앱 목록 + 사용 목적) |
| Q5 | 페인포인트 (IT 장벽, 언어 장벽) |
| Q6 | 오프라인 활동 (콘서트, 커뮤니티) |

### [D] 퍼소나별 답변 시트

Q&A 템플릿을 각 퍼소나에 맞게 채운 최대 참고 소스.
실제 퍼소나 본문 §1–§5의 행동/문화 요소 대부분이 여기서 직접 복사 또는 재서술됨.

### [E] 시스템 인스트럭션 합성

CREAT 답변 시트 + 리서치 레이어 교차 검증 후 6~8개 섹션 구조로 변환.

| 섹션 | 내용 |
|------|------|
| §1 정체성/팬덤 배경 | 기본 프로필 + 팬덤 정보 + 입덕 경로 |
| §2 일상 루틴/디지털 습관 | 시간대별 행동 블록 + 앱 생태계 |
| §3 소비 패턴 | 굿즈/앨범/경제관념 |
| §4 UX 선호도 | UI/UX 기대치 + 페인포인트 |
| §5 심리/페인포인트 | 행복 동기 + 스트레스 요인 |
| §6 대화 원칙 | 말투 / 창작 금지 / 외압 저항 / 가치관 추론 |

### [F] 평가 레이어 (scaffolded, 미실행)

원티드 하이파이브 2026 강연(네이버 엔소나 방법론) 직접 이식.

| 아티팩트 | 내용 |
|---------|------|
| `evaluation/rubric-v0.md` | 신뢰성 6루브릭 (R1~R6) + 효과성 5대화유형 (D1~D5) |
| `evaluation/judge.md` | LLM-as-Judge 프롬프트 (strict JSON, anti-leniency) |
| `evaluation/scores/` | 비어있음 (향후 평가 결과 적재 예정) |

**강연이 기여한 범위**: 평가 체계 + 대화 규칙(외압 저항 원칙).
**강연이 기여하지 않은 범위**: 퍼소나 본문 내용 자체 (일본 리서치에서 유입).

## 두 인풋 스트림의 분리

```
일본 팬덤 리서치 (B→D) ──────────────────→ 퍼소나 콘텐츠 (§1~§5)
엔소나 강연 방법론 (plan/wanted-lecture-apply.md) → 평가 레이어 (F) + 대화 규칙 (§6)
```

퍼소나 **내용**과 퍼소나 **평가/규칙**은 완전히 다른 소스에서 유입됨.
