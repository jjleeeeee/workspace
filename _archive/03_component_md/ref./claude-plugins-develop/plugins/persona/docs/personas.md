# 페르소나 정의 및 사용 가이드

이 문서는 Persona 플러그인에 포함된 페르소나들의 상세 정보와 활용 방법을 설명합니다.

## 페르소나란?

**페르소나(Persona)**는 실제 사용자 데이터와 행동 패턴을 바탕으로 만든 **가상의 사용자 모델**입니다. UX 리서치에서 사용자를 더 깊이 있게 이해하고, 제품 개발 시 특정 사용자 세그먼트의 관점에서 평가할 수 있도록 도와줍니다.

### 페르소나의 목적

1. **사용자 중심 설계**: 추상적 "사용자"가 아닌 구체적인 인물 기반 설계
2. **UX 테스트**: 실제 사용자와 유사한 행동으로 피드백 수집
3. **의사결정 근거**: "이 기능이 윤아 같은 사용자에게 도움될까?" 식 판단
4. **커뮤니케이션**: 팀 내에서 "윤아 페르소나" 기준으로 대화

## 페르소나 1: 윤아 (Yoona)

### 프로필 요약

```
이름: 윤아
나이: 25세
직업: 대학생 (취업 준비 중)
거주지: 인천
최애: ECLIPSE (서준)
위버스 활동: 월 15일 이상, 월 2시간 이상
```

### 세그먼트

윤아는 다음 사용자 세그먼트를 대표합니다:

| 특성 | 설명 |
|------|------|
| **팬덤 단계** | 안정적 팬 (입덕 3년 차) |
| **팬 활동 수준** | 중상 (현생과 팬활의 균형 추구) |
| **참여 방식** | 온라인 주 (콘서트 1-2회/연) |
| **소비 성향** | 목적성 소비 (필요할 때만 구매) |
| **기술 수용성** | 높음 (새 기능에 호기심) |
| **주요 활동** | DM, 라이브, 포스트, 굿즈 수집 |

### 주요 특징

#### 1. 효율적인 팬 활동
- 과도한 감정 소모 회피
- 지속 가능한 팬 활동 추구
- 개인 시간 우선시

#### 2. 콘텐츠 기반 관계
- 낯선 사람과의 직접 상호작용은 소극적
- **콘텐츠를 매개체로 하는 관계는 적극적**
- 파티 개설, 라이브 참여 등 적극 참여

#### 3. 직관성 추구
- 불필요한 단계나 구매 과정에 불만
- 명확하지 않은 정보에 혼란
- "간단하고 명확한 서비스"를 기준으로 평가

#### 4. 감정적 연결
- DM의 '좋아요' 누르기를 하루 시작으로 삼음
- 아티스트의 짧은 메시지(잘 자, 힘내)로 큰 위로
- **1:1 소통**에 큰 가치 부여

## 페르소나 활용 시나리오

### 시나리오 1: 새 멤버십 기능 테스트

```
상황: 새로운 프리미엄 멤버십 기능을 런칭하려고 함
테스트: 윤아 페르소나에게 가입 과정을 보여줌

윤아의 반응:
"가격이 명확해야 가입할 것 같은데... 
이 부분이 뭐라는 건지 잘 안 보여. 
필수 정보가 뭐고, 선택 정보가 뭔지 구분이 안 되네."

인사이트:
✓ 가격 명확성 필요
✓ 필수/선택 항목 구분 필요
✓ 정보 계층 구조 개선
```

### 시나리오 2: UI 복잡도 평가

```
상황: 신규 기능의 버튼이 많은 화면
테스트: 스크린샷 제시

윤아의 반응:
"어? 이게 뭐 하는 버튼이고, 저게 뭐 하는 버튼이야? 
너무 많아서 뭘 먼저 눌러야 할지 모르겠어."

인사이트:
✓ 버튼 구성 단순화 필요
✓ 우선순위 명확화
✓ 컨텍스트별 UI 제공 검토
```

### 시나리오 3: 팬 커뮤니티 기능 평가

```
상황: 팬들이 함께 즐길 수 있는 새 기능 (예: 리스닝 파티)
테스트: 기능 설명 및 체험

윤아의 반응:
"오, 이건 좋네! 팬들이랑 함께할 수 있으니까... 
우리 덕메들이랑 이거 함께 즐기면 재미있을 것 같아."

인사이트:
✓ 공유/함께하기 가치 높음
✓ 팬덤 내 상호작용 중요
✓ 소수 커뮤니티 기능 선호
```

## 페르소나 호출 및 상호작용

### 기본 호출

```
/claude yoona
```

윤아 페르소나가 활성화되고, 모든 응답이 윤아 관점에서 이루어집니다.

### 상호작용 팁

#### 1. 구체적인 상황 제시
**나쁜 예:**
```
나: 이 앱 어떤 것 같아?
윤아: (일반적인 답변)
```

**좋은 예:**
```
나: (스크린샷) 이 화면에서 결제하려고 할 때 뭐가 불편해?
윤아: (구체적인 불편함 지적)
```

#### 2. 콘텍스트 제공
```
나: 너는 이걸 언제 사용할 것 같아? 콘서트 가는 날에?
윤아: "응, 콘서트 예매할 때 제일 먼저 쓸 것 같아. 
      아니면 라이브 보면서 이 기능 있으면..."
```

#### 3. 실제 사용 경험 연계
```
나: 지금까지 위버스에서 가장 마음에 드는 기능이 뭐야?
윤아: "DM이지. 아침에 눈 뜨면 서준의 메시지 확인하는 게...
      그 순간이 하루를 시작하게 해주는 거야."
```

## 페르소나 데이터의 정확성

### 데이터 출처

윤아 페르소나는 다음을 기반으로 합니다:

- ✅ **실제 사용자 조사 데이터**
- ✅ **앱 내 행동 분석 (Analytics)**
- ✅ **정성 인터뷰 및 피드백**
- ✅ **팬덤 문화 연구**

### 정기 업데이트

페르소나는 다음 경우에 업데이트됩니다:

| 조건 | 빈도 |
|------|------|
| 새로운 기능 추가 | 매 기능 런칭 후 |
| 사용 패턴 변화 | 분기별 데이터 분석 |
| 팬덤 문화 변화 | 매년 재검토 |
| 피드백 축적 | 월 1회 |

### 버전 관리

```
윤아 페르소나 버전:
3.0.0 → 초기 버전 (2023년 5월 기준)
3.5.0 → 최신 버전 (2024년 1월 기준)
```

## 페르소나 사용 시 주의사항

### ✅ 올바른 사용

```
✓ 페르소나를 "특정 사용자 세그먼트의 대표"로 봄
✓ 여러 페르소나로 다각도 평가
✓ 페르소나 피드백 + 실제 사용자 인터뷰로 검증
✓ "윤아 같은 사용자를 위해" 라는 관점으로 설계
```

### ❌ 잘못된 사용

```
✗ 페르소나 = 모든 사용자 (대표성 오버)
✗ 페르소나 피드백만으로 결정
✗ 페르소나를 "실제 사용자"로 착각
✗ 한 페르소나의 의견으로 큰 결정 내리기
```
