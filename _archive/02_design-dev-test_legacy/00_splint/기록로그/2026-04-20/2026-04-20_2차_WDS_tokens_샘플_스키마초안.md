## 발견/결과

### Figma Variable 1개당 획득 가능한 필드

`id`, `name`, `key`, `resolvedType`, `scopes`, `codeSyntax{ANDROID,iOS}`, `description`, `hiddenFromPublishing`, `valuesByMode` (alias 참조), `resolvedValuesByMode{light,dark}` (raw + aliasTo)

### 샘플 데이터 shape (system/color/text/primary)

```json
{
  "id": "VariableID:491:7017",
  "name": "system/color/text/primary",
  "key": "cd617200247f96107193d8319f1160d0f240f656",
  "resolvedType": "COLOR",
  "scopes": ["SHAPE_FILL", "TEXT_FILL"],
  "codeSyntax": { "ANDROID": "textPrimary", "iOS": "textPrimary" },
  "description": "",
  "hiddenFromPublishing": false,
  "valuesByMode": {
    "490:0": { "type": "VARIABLE_ALIAS", "id": "VariableID:7602:99931" },
    "7606:6": { "type": "VARIABLE_ALIAS", "id": "VariableID:7606:100191" }
  },
  "resolvedValuesByMode": {
    "light": { "value": "#00B8C1", "aliasTo": "base/color/light/mint-600" },
    "dark":  { "value": "#01D5DF", "aliasTo": "base/color/dark/mint-500" }
  }
}
```

### 핵심 관찰

- **`scopes` 필드 존재:** 각 토큰에 Figma가 이미 `SHAPE_FILL`, `TEXT_FILL`, `ALL_SCOPES` 등 usage를 태깅해둠 → semantic usage hint로 즉시 활용 가능
- **`codeSyntax` 필드 존재:** ANDROID/iOS 플랫폼별 변수명 이미 정의됨 (예: `textPrimary`) → 플랫폼 DSL→Code 변환의 자동 근거. **단, WEB 항목 없음**
- **`resolvedValuesByMode` 제공:** alias 체인을 서버에서 이미 해석 → fixture 생성 시 `base/color/light/mint-500` 같은 참조를 별도 resolve할 필요 없음. 단 `aliasTo` 필드로 투명성 유지
- **`description` 전부 비어있음:** 9개 샘플 모두 빈 문자열 — 토큰 문서화 공백
- **의미 중복 발견:** `system/color/roles/primary` · `system/color/roles/brand-mint` · `system/color/button/default` 모두 `base/color/light/mint-500` → `#00CBD5` (light) 동일. LLM이 "어느 걸 써야 하나" 혼동 유발 가능 → diagnostics 후보

## Token Catalog Schema v1 초안

### 전체 구조

```json
{
  "$schema": "https://our-ds-mcp/schemas/tokens.v1.json",
  "source": {
    "origin": "figma:Chord Design System",
    "fileKey": "DWEduE6GfxYMlyxKPNJ8jA",
    "collection": "WDS_tokens",
    "version": "wds_version:<base/wds_version value>",
    "freshness": "2026-04-20T09:34Z",
    "coverage": { "total": 753, "documented": 0, "hidden": 0 },
    "stageLabel": "v0.1-draft"
  },
  "modes": ["light", "dark"],
  "tokens": [
    {
      "id": "token:system.color.text.primary",
      "name": "system/color/text/primary",
      "type": "color",
      "values": {
        "light": { "raw": "#00B8C1", "aliasOf": "base/color/light/mint-600" },
        "dark":  { "raw": "#01D5DF", "aliasOf": "base/color/dark/mint-500" }
      },
      "usage": ["shape-fill", "text-fill"],
      "platformCode": {
        "ios":     "textPrimary",
        "android": "textPrimary",
        "web":     "text-primary"
      },
      "description": null,
      "hidden": false,
      "figmaKey": "cd617200247f96107193d8319f1160d0f240f656",
      "diagnostics": ["documentation:missing"]
    }
  ],
  "diagnostics": [
    "duplicate-alias:3 tokens aliased to base/color/light/mint-500 — consider consolidation"
  ]
}
```

### 설계 결정

| 요소 | 결정 | 근거 |
|---|---|---|
| ID 체계 | `token:<dot-path>` | Figma VariableID는 세션/파일 이동 시 불안정, dot-path가 플랫폼 중립 |
| 타입 enum | `color` · `dimension` · `number` · `string` | Figma COLOR/FLOAT/STRING 매핑, FLOAT은 context별 dimension/number로 세분 |
| values 구조 | `{ [mode]: { raw, aliasOf } }` | resolved 실제 값과 alias 체인 둘 다 보존 — LLM이 의미 추론 가능 |
| usage enum | `shape-fill`, `text-fill`, `stroke`, `effect`, `all` | Figma `scopes`를 semantic 표기로 재명명 |
| platformCode | `{ ios, android, web }` | ios/android는 codeSyntax에서 직접, web은 자동 파생(kebab-case) 또는 오버라이드 |
| per-token `diagnostics` | 배열 | 문서화 누락, 중복 alias 등 per-token 표시 |
| catalog `source` 메타 | 필수 | 인터페이스 3원칙 준수 (source 메타필드 필수) |
| catalog `diagnostics` | 배열 | 전역 경고 (중복 alias 그룹, 커버리지 등) |
