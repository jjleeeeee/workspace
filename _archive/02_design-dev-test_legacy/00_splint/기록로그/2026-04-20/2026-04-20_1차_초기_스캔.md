# 2026-04-20 — 1차 figma-console MCP 스캔 (Chord Design System)

> 상위 문서: [../figma_ds_structure_audit_260420.md](../figma_ds_structure_audit_260420.md)

**목표:** Chord DS Figma 파일에 figma-console MCP로 연결하고, Token/Component 구조 초기 스냅샷 확보.

## 사용 도구/방법

- `figma_get_status` (probe)
- `figma_get_design_system_summary`
- `figma_get_variables` (format=summary, verbosity=summary)
- `figma_search_components` (query="Button", limit=15)
- 대상 파일: `Chord Design System` (fileKey `DWEduE6GfxYMlyxKPNJ8jA`)

## 진행 과정

1. Desktop Bridge 플러그인 연결 시도 → 첫 연결 성공 후 즉시 드롭
2. 중복 figma-console-mcp 프로세스 (pid 1503 외)가 9223 포트 점유 확인 → `kill` 후 잔여 프로세스(47720, 49215)까지 `kill -9`로 정리
3. Claude Desktop의 Figma PAT 갱신 — `claude_desktop_config.json`의 `FIGMA_ACCESS_TOKEN` 전역 교체 + Claude Desktop 재시작
4. Figma Desktop Bridge 재실행 → port 9225(fallback)로 안정 연결
5. 3개 tool 스캔 실행 (summary → variables → Button 컴포넌트)

## 실패 및 원인

- **1차 스캔 일제 실패:** 세 tool 동시 병렬 실행 시 `get_design_system_summary` 타임아웃 + 나머지 2개는 "connection lost". 원인 = 포트 9223에 중복 MCP 인스턴스 점유, 플러그인이 9223/9224 사이에서 재접속하다 드롭.
- **Desktop Bridge 플러그인 연결 불안정:** PAT 갱신 전 상태에서는 상태 조회(status probe)는 OK였으나 tool 호출 시점에 연결이 끊겨 있음.

## 해결 방법

- 중복 figma-console-mcp 프로세스 전부 `kill -9`
- Claude Desktop config의 `FIGMA_ACCESS_TOKEN` 일괄 교체 (jq 사용) + Claude Desktop 재시작
- Figma Desktop Bridge 플러그인 재실행 → 안정 연결 확보

## 다음 사이클 이슈

1. **네이밍 inconsistency 목록화 (diagnostics 후보):**
   - `Text Button` vs `Text button` (대소문자)
   - `_atoms` vs `_atoms ` (trailing space)
   - Button 12개 중 "공식" 단일화 필요
2. **`_` prefix 컴포넌트 취급 정책** — `_atoms`, `_Keys`, `_Toolbar` 등 내부용으로 보이는 컴포넌트를 DS-MCP 카탈로그에서 제외할지 포함할지 결정
3. **TYPO_tokens의 플랫폼별 mode(iOS/AOS/WEB) 표현 방식** — Catalog Schema에 platform-specific 필드를 어떻게 둘지 설계 필요 (플랫폼 중립 원칙과 충돌)
4. **`Text Button`(576 variants) props 축 추출** — `figma_analyze_component_set`로 variant 축을 뽑아 Component Catalog v1의 Button 스키마 초안 작성
5. **`tag` 컴포넌트 description 형식**을 Component Catalog Schema v1 템플릿으로 채택 검토
6. **Collection별 full dump** — `WDS_tokens` 753개 전수 조사해 `fixtures/tokens.sample.json` 초안 준비
