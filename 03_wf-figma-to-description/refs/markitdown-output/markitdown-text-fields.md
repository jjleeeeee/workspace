# [Usage] Text Fields

Text Fields는 사용자가 텍스트를 입력하거나 편집할 수 있도록 하며, 사용자로부터 정보를 수집하기 위해 사용하는 입력 컴포넌트입니다.

## Context

Text Fields는 폼, 검색창, 다이얼로그, 인증, 댓글 입력처럼 사용자의 직접 입력이 필요한 상황에서 사용해야 합니다. 사용자가 어떤 정보를 입력해야 하는지, 입력 상태가 정상인지, 오류가 있는지를 즉시 이해할 수 있어야 합니다. 간단한 입력창은 Single을 사용하고, 2줄 이상 입력 텍스트 값이 노출되는 경우 Multiple 또는 Multi-line을 사용해야 합니다.

## Principle

- 입력 목적을 이해하기 위해 라벨과 힌트를 명확하게 제공해야 합니다.
- 입력 상태를 구분하기 위해 Status와 피드백을 일관되게 사용해야 합니다.
- 입력 길이와 목적에 따라 Single과 Multiple을 구분해야 합니다.
- 모든 사용자가 쉽게 사용할 수 있도록 키보드 탐색과 스크린 리더 지원을 고려해야 합니다.

## Usage Do / Don't

### Label

Do

- 사용자가 입력해야 할 정보를 이해하기 위해 라벨을 제공해야 합니다.

Don't

- Placeholder만으로 필드 목적을 설명하지 않아야 합니다. (입력 후 맥락 소실)

### Status

Do

- Default, Focused, Error, Disabled 등 각 Status에 맞는 값을 사용해야 합니다.

Don't

- 하나의 필드에 여러 Status 값을 섞어 적용하지 않아야 합니다. (상태 의미 혼동)

### Feedback

Do

- 오류나 유효성 검사가 필요한 경우 즉각적인 피드백 문구를 제공해야 합니다.

Don't

- 오류 상태를 컬러만으로 전달하지 않아야 합니다. (접근성 저하)

### Lines

Do

- 간단한 입력창은 Single을, 2줄 이상 입력 텍스트 값 노출 시 Multiple 또는 Multi-line을 사용해야 합니다.

Don't

- 여러 줄 입력이 필요한 내용을 Single에 넣지 않아야 합니다. (내용 확인 어려움)

### Accessibility

Do

- 키보드 탐색과 스크린 리더가 입력 목적, 상태, 오류 피드백을 이해할 수 있어야 합니다.

Don't

- 라벨, 포커스 상태, 오류 문구를 보조 기술에서 확인할 수 없게 만들지 않아야 합니다. (접근성 저하)

### Action

Do

- 삭제, 보기, 재전송 같은 보조 액션은 입력 목적과 직접 관련될 때만 제공해야 합니다.

Don't

- 입력과 무관한 액션을 필드 내부에 배치하지 않아야 합니다. (조작 혼동)
