# [Usage] Tabs

Tabs는 한 화면에서 다양한 카테고리의 화면을 빠르게 이동하고, 관련 콘텐츠를 즉시 전환하기 위해 사용하는 컴포넌트입니다.

## Context

Tabs는 사용자가 현재 화면을 벗어나지 않고 유사한 카테고리나 관련 있는 그룹을 탐색해야 할 때 사용해야 합니다. 탭을 클릭하면 하단 또는 내부의 해당 콘텐츠가 즉시 표시되어야 하며, 타입은 Chips와 Bar 맥락을 구분해 적용해야 합니다. 앱의 최상위 이동은 Bottom Navigation을 사용하고, 서로 다른 작업 흐름이나 독립 페이지 이동에는 Tabs를 사용하지 않아야 합니다.

## Principle

- 유사한 카테고리나 관련 있는 그룹을 모아 공간을 효율적으로 활용해야 합니다.
- 선택된 탭은 활성화되어 있음을 시각적으로 알려 사용자의 위치를 표시해야 합니다.
- 탭을 클릭하면 하단 또는 내부의 해당 콘텐츠가 즉시 표시되어야 합니다.
- 같은 탭 그룹 안에서는 Type, Size, Padding 규칙을 일관되게 적용해야 합니다.

## Usage Do / Don't

### Type

Do

- 성격이 다른 Chip을 제외하고 같은 탭 그룹 안에서는 동일한 Type을 사용해야 합니다.

Don't

- Img와 Text 타입을 같은 탭 그룹 안에 혼용하지 않아야 합니다. (선택 기준 혼동)

### 콘텐츠 전환

Do

- 탭을 클릭하면 하단 또는 내부의 해당 콘텐츠가 즉시 표시되어야 합니다.

Don't

- 탭 클릭을 관련 없는 화면 이동이나 독립 페이지 전환으로 연결하지 않아야 합니다. (탭 탐색 맥락 이탈)

### 긴 레이블

Do

- 최대 길이를 넘어갈 경우 Marquee를 적용해야 합니다.

Don't

- 긴 탭 레이블을 말줄임표로 처리하지 않아야 합니다. (카테고리 식별 정보 손실)

### Size

Do

- 이미지가 들어가는 경우에만 Medium 사이즈를 사용해야 합니다.

Don't

- Text 또는 Icon Type으로 사용될 때 Medium 사이즈를 사용하지 않아야 합니다. (시각 밀도 불균형)

### Padding

Do

- 공통된 좌우 Padding 값을 사용해야 합니다.

Don't

- Padding 값을 임의로 변경하지 않아야 합니다. (터치 영역 및 정렬 일관성 저하)

### 선택 상태

Do

- 선택된 탭은 활성화되어 있음을 시각적으로 알려 사용자의 위치를 표시해야 합니다.

Don't

- 선택된 탭과 선택되지 않은 탭을 같은 스타일로 노출하지 않아야 합니다. (현재 위치 식별 어려움)
