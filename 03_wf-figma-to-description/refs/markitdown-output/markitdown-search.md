# [Usage] Search

Search는 사용자가 키워드를 입력해 원하는 정보나 결과를 찾기 위해 사용하는 입력 컴포넌트입니다.

## Context

Search는 목록, 주소, 콘텐츠, 사용자처럼 사용자가 직접 조건을 입력해 결과를 좁혀야 할 때 사용해야 합니다. 일반 Text Field와 달리 검색 아이콘, 입력 중 상태, 전체 삭제 액션, 결과 피드백이 함께 제공되어야 합니다.

## Principle

- 사용자가 검색 입력임을 인지하기 위해 검색 아이콘을 앞에 노출해야 합니다.
- 검색어 입력 중에는 전체 삭제 버튼을 제공해야 합니다.
- 결과 수나 결과 없음 상태를 검색 입력과 연결해 보여줘야 합니다.

## Usage Do / Don't

### Icon

Do

- 검색 목적을 알리기 위해 검색창 앞에는 검색 아이콘을 노출해야 합니다.

Don't

- 검색 아이콘 없이 일반 입력 필드처럼 보이게 만들지 않아야 합니다. (기능 인지 저하)

### Clear Button

Do

- 검색어 입력 중에는 우측에 전체 삭제 버튼을 노출해야 합니다.

Don't

- 입력된 검색어가 있는데 삭제 액션을 숨기지 않아야 합니다. (수정 비용 증가)

### Keyboard

Do

- 검색창 클릭 시 키보드가 올라오도록 해야 합니다.

Don't

- 입력 가능한 검색창을 비활성 상태처럼 보이게 만들지 않아야 합니다. (조작 가능성 혼동)

### Result

Do

- 검색 결과가 있는 경우 결과 수와 목록을 검색어와 가까운 맥락에서 보여줘야 합니다.

Don't

- 검색 결과와 입력 필드의 관계를 떨어뜨려 배치하지 않아야 합니다. (결과 맥락 혼동)
