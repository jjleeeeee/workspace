## 배경

checkout.md v3 초안 작성 후, 디자인 스펙(수치)과 컴포넌트 참조가 빠져 있어 실제 구현에 부족하다는 문제 인식. figma-console JS로 자동 추출하는 방식을 실험함.

---

## 자동 추출한 것들

### 1. 레이아웃 스펙 (JS로 추출)

```js
// paddingTop/Bottom/Left/Right, itemSpacing, cornerRadius, fills 추출
const layout = getLayout(node);
```

| 항목 | 추출 방식 | 결과 |
| --- | --- | --- |
| 섹션 카드 padding | node.paddingTop/Left 등 | 16px 전체 |
| 섹션 간 간격 | body.itemSpacing | 12px |
| Body 좌우 여백 | body.paddingLeft/Right | 10px |
| GNB 높이 | gnb.height | 98px |
| Bottom 높이 | bottom.height | 336px |
| 카드 borderRadius | node.cornerRadius | 16px |

**발견**: 주문 상품 섹션만 paddingLeft/Right=0 (상품 이미지 카드 끝까지 확장).

### 2. 섹션별 컴포넌트 인스턴스 (JS로 추출)

```js
// INSTANCE 타입 노드에서 mainComponent.parent(ComponentSet) 이름 추출
async function getInstances(node) { ... }
```

아이콘(24/em/*, 16/em/*), 내부 atom(_atoms/*, element/*)은 제외하고 디자인 시스템 컴포넌트만 매핑.

## 워크플로우 정리

화면 MD 작성 시 자동화 가능한 단계:

```
1. figma-console JS → 레이아웃 수치 추출 (padding, spacing, radius, 높이)
2. figma-console JS → 섹션별 INSTANCE 목록 추출
3. 아이콘/atom 필터링 → 디자인 시스템 컴포넌트만 추림
4. component MD 존재 여부 확인 → 링크 연결 또는 "MD 미작성" 표기
5. 사람이 의미 없는 항목 정리 (5분 이내)
```

REST API 전체 읽기 대비 토큰 소비 대폭 감소. 구조 파악 + 수치 추출 모두 JS로 처리 가능.
