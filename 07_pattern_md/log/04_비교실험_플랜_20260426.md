## 실험 설계

### 공통 프롬프트
```
이 MD를 기반으로 결제하기 화면을 React Native로 구현해줘
```

### 실험 A — 단일 파일
AI에게 넘기는 파일:
- `checkout_v1_fullpage_inline.md`
- `button.md`, `checkbox.md`

### 실험 B — 섹션 분리
AI에게 넘기는 파일:
- `checkout/index.md` + 섹션 파일 전체 (8개)
- `button.md`, `checkbox.md`

---

## 평가 기준

| 항목 | 체크 포인트 |
| --- | --- |
| 구조 정확도 | GNB Fixed, Body Scroll, Bottom Fixed 구현 여부 |
| 컴포넌트 사용 | 섹션별 컴포넌트 props가 MD 기준과 일치하는가 |
| 상태 처리 | CTA Disabled/Enabled, Accordion 구현 여부 |
| 누락 여부 | MD에 있는데 구현 안 된 섹션 |
| 토큰 효율 | AI가 읽은 파일 수/분량 |

---

## 선행 결정 필요 사항

실험 전에 결정해야 할 것:

| 질문 | 선택지 |
| --- | --- |
| 컴포넌트 MD 완성도 | A. 지금 바로 실험 (미작성 컴포넌트는 AI 재량) / B. 주요 MD 먼저 작성 (Radio, Text_Fields, Dropdown) / C. 전체 완성 후 실험 |
| 구현 타겟 | React Native / React (Web) / 피그마 |

→ **"파일 구조"를 먼저 검증**하려면 A 선택 (지금 바로 실험 가능)
→ **"MD → 구현 가능성"을 검증**하려면 B/C 선택 (컴포넌트 MD 작성 선행)
