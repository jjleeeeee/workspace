## 화면 MD에서 컴포넌트 기술 방식

### 결론: 화면에서의 설정값만 인라인, 규격은 참조

```markdown
Button: Type=Outlined_gray, Size=XSmall(32), label="변경"
→ [button.md](../../99_archive/03_component_md/components/button.md)
```

- 인라인: 이 화면에서 실제 사용되는 값 (Type, Size, label 등 핵심만)
- 참조: trailing 동작, 토큰, 상태 매트릭스 등 컴포넌트 규격은 button.md에 위임
- AI가 trailing 같은 세부 동작이 궁금하면 button.md를 참조하면 됨

### Figma 내부 props를 그대로 쓰지 않는다

피그마 componentProperties 예시:
```
Option-Trailing: false
-> Trailing-Icon: true   ← Option-Trailing=false면 렌더링 안 됨
```

`-> ` 접두어가 붙은 props는 부모 prop이 false면 무시됨.
이런 계층 구조를 MD에 그대로 쓰면 오해 발생 가능.

→ **렌더링 결과 기준으로 기술.** 피그마 내부 prop 구조 노출 X.

```markdown
# 나쁜 예
Button: Option-Trailing=false, ->Trailing-Icon=true

# 좋은 예
Button: Type=Outlined_gray, Size=XSmall(32), label="변경"
```

## props 기술 깊이 원칙

| 기술 대상 | 화면 MD에 포함 | 이유 |
| --- | --- | --- |
| Type, Size, label 등 핵심 variant | O | 화면마다 다름 |
| 상태 매트릭스 (Disabled, Loading 등) | X | button.md에 있음 |
| 토큰, 접근성, 높이 수치 | X | button.md에 있음 |
| 피그마 내부 구조 props (Option-*, ->*) | X | 렌더링 결과로 대체 |

**목표: "읽었을 때 틀리지 않는 수준"이지 완벽한 정확도가 아님.**

## 자동화 가능성 (미탐색)

figma-console JS로 componentProperties에서 의미 있는 props만 자동 추출 가능.

```js
Object.entries(node.componentProperties)
  .filter(([k, v]) => v.value !== false)
  .map(([k, v]) => `${k.replace(/^-> /, '')}=${v.value}`)
```

→ 화면 MD 초안 자동 생성 후 사람이 경량 정리하는 방식으로 운영 가능. 추후 실험 예정.
