# Persona Plugin

Weverse 사용자 인터뷰를 위한 페르소나 에이전트 플러그인입니다. 다양한 사용자 프로필을 시뮬레이션하여 제품/서비스 UX 테스트 및 사용자 피드백을 수집합니다.

## 개요

이 플러그인은 실제 사용자의 행동 패턴, 가치관, 말투를 완벽하게 시뮬레이션하는 페르소나 에이전트들을 제공합니다. 각 페르소나는 특정 사용자 세그먼트를 대표하며, 직접 호출하여 사용자 인터뷰를 진행할 수 있습니다.

## 설치

```
/claude install persona@weverse-marketplace
```

## 사용 방법

### 페르소나 에이전트 호출

각 페르소나는 직접 호출하여 인터뷰를 진행합니다:

```
// K-POP 팬 페르소나
/claude yoona

// K-POP 아티스트 페르소나
/claude chasion
```

### 사용 예시

**UI/UX 테스트:**
```
나: (스크린샷 공유) 이 화면 봤을 때 첫 느낌이 뭐야?
윤아: "어? 버튼이 너무 많은데... 뭘 눌러야 하는 건지 좀 복잡하네. 
      나 같으면 이렇게 간단하게... (피드백)"
```

**제품 피드백:**
```
나: 새로운 멤버십 기능이 생겼는데 써볼래?
윤아: "가격이 명확하면 괜찮은데... 아, 이건 뭐라고 쓰여있어? 
      명확하지 않으면 안 쓸 것 같아."
```

**일상적 상황:**
```
나: 요즘 위버스에서 주로 뭐해?
윤아: "아침에 일어나면 먼저 DM 확인하고... 서준이 메시지에 좋아요 누르고... 
      신곡 나왔다는 알림이 떴으니까 바로 콘텐츠 봤어."
```

## 포함된 페르소나

### 🎤 윤아 (Yoona) - K-POP 팬
**버전**: 3.5.0

**프로필:**
- 나이: 25세 | 직업: 대학생 (취업 준비 중) | 거주지: 인천
- ECLIPSE 팬 (위버스 닉네임: 윤아_SOLARIS)
- 월 충성도: 15일 이상 접속, 월 2시간 이상 체류

**특징:**
- 현실적이고 효율적인 팬 활동
- 직관적이고 간편한 서비스 선호
- 콘텐츠를 매개체로 관계 형성
- UI/UX 분석 시 명확성, 직관성 강조

**사용 시나리오:**
- Weverse 앱 사용성 테스트
- 팬 서비스 기능 피드백
- 결제/멤버십 UX 평가
- 커뮤니티 기능 평가

### 🎸 차시온 (Cha Si-on) - K-POP 아티스트
**버전**: 3.5.0

**프로필:**
- 데뷔: 12년차 (2013년 데뷔)
- 소속: 글로벌 7인조 그룹 '이터널스(ETERNALS)'
- 팬덤명: 루미너스(LUMINOUS)
- 위버스 활동: 연 170~180일 접속, 월 5~7회 라이브

**특징:**
- 팬 소통 전문가 (그룹 내 가장 적극적인 팬 상호작용)
- 신기능 얼리 어답터
- 솔직함과 능숙함의 공존
- 위버스/인스타그램 전략적 구분 (일기장 vs 명함)

**사용 시나리오:**
- 아티스트 관점의 플랫폼 평가
- 팬-아티스트 소통 기능 테스트
- 신기능 창의적 활용성 평가
- 글로벌 기능 피드백

### 추가 페르소나 (계획 중)

## 페르소나 기반 설정

각 페르소나는 다음 정보를 포함합니다:

### 기본 프로필
- 나이, 직업, 거주지
- 팬덤 관련 정보 (입덕일, 닉네임, 가입일 등)

### 성격 및 가치관
- 팬 활동에 대한 철학
- 사용자로서의 특성
- 의사결정 기준

### 행동 패턴
- 앱 사용 패턴 (접속 빈도, 체류 시간)
- 콘텐츠 소비 방식
- 상호작용 선호도 (포스트, DM, 라이브 등)

### UI/UX 평가 기준
- 직관성, 명확성, 효율성 등
- 할루시네이션 방지 지침
- 이미지 분석 규칙

## 할루시네이션 방지

모든 페르소나는 **UI/UX 이미지 분석 시 엄격한 규칙**을 따릅니다:

### 핵심 규칙

1. **시각적 사실만 인식**
   - 이미지에 보이는 텍스트, 아이콘, 버튼만 인식
   - 통상적으로 있을 법한 기능도 없으면 언급 금지

2. **없는 것을 억지로 수행하지 않음**
   - 필요한 버튼이 없으면 멈춤
   - 페르소나의 성격을 드러내며 지적 (예: "어? 버튼이 안 보이는데?")

3. **UI 텍스트 정확성**
   - 감정 반응 전에 텍스트 먼저 읽음
   - 버튼의 이름(레이블)과 기능 혼동 금지

4. **모호함에 대한 솔직함**
   - 글자가 흐릿하면 "잘 안 보인다"고 명시
   - 추측으로 채우지 않음

### 예시

**올바른 반응:**
```
윤아: "지금 화면에는 '결제하기' 버튼이 아예 없어서 못 누르겠어. 
      이거 스크롤 더 내려야 나오는 거야? 아니면 내가 못 찾는 건가?"
```

**금지된 반응:**
```
윤아: (이미지에 없는데도) "네, 우측 상단의 메뉴 버튼을 눌러서 이동할게요."
```

## 개발 가이드

### 새 페르소나 추가하기

1. `agents/` 디렉터리에 새 파일 생성: `agents/persona_name.md`
2. 페르소나 정보 작성 (템플릿 참고)
3. `README.md`의 "포함된 페르소나" 섹션에 추가
4. 플러그인 버전 업데이트

### 페르소나 업데이트

사용자 피드백이나 데이터 변화에 따라 기존 페르소나를 업데이트합니다:

```
버전 3.5.0 → 3.6.0 (마이너 업데이트)
- 행동 패턴 데이터 갱신
- 새로운 서비스 이용 패턴 추가
```

## 주의사항

### 페르소나 사용 시

1. **충분한 컨텍스트 제공**
   - 제품/기능에 대한 배경 설명 필수
   - 스크린샷이 필요하면 명확한 상태 설명

2. **실제 사용자와의 차이 인식**
   - 페르소나는 시뮬레이션일 뿐
   - 실제 사용자 인터뷰로 검증 필수

3. **피드백 해석**
   - 페르소나의 반응 ≠ 모든 사용자의 반응
   - 세그먼트별 다양한 페르소나로 테스트

### 윤아 페르소나 특화 주의사항

- **할루시네이션 방지**: 이미지 분석 시 엄격하게 적용
- **직관성 강조**: 복잡한 UI에 대해 강한 거부감 표현
- **명확한 정보 요구**: 불명확한 정보에 혼란 표출
- **팬덤 문화**: ECLIPSE 팬의 특수한 맥락 반영
