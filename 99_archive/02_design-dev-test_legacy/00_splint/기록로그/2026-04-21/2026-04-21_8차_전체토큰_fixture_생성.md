**목표:** 3개 Figma 토큰 컬렉션 전체를 `schemas/tokens.v1.json` 형식의 fixture 파일로 추출한다. 이후 컴포넌트 카탈로그에서 tokenBindings.token 값을 이 fixture와 cross-join하면 "토큰 이름 조회" 단계가 제거된다.

## 사용 도구/방법

- figma-console MCP `figma_get_variables(format=filtered, collection=*, resolveAliases=true, verbosity=standard, pageSize=100)`
- 대용량 응답 → `/tmp/raw_*.json` 자동 저장 후 `jq` 추출
- `scripts/figma-to-tokens-v1.mjs` (신규 작성) — 배치 JSON → v1 fixture 변환
- `scripts/validate-tokens.mjs` — ajv 검증

### 2. `scripts/figma-to-tokens-v1.mjs` 작성

주요 매핑 규칙:
- `name` → `id`: `"token:" + name.toLowerCase().replace(/\//g, '.').replace(/_/g, '-')`
- `resolvedType=COLOR` → `type: "color"`
- `resolvedType=FLOAT` + FONT_WEIGHT scope → `type: "number"`
- `resolvedType=FLOAT` + 그 외 → `type: "dimension"`
- `resolvedType=STRING` → `type: "string"`
- 모드명 정규화: `iOS`→`ios`, `AOS`→`aos`, `WEB`→`web`, `Mode 1`→`mode-1`
- `key || id` → `figmaKey`
- description 공란 → null
- catalog diagnostics: `documentation:missing`, `platform-code:missing-web`

### 3. 스키마 패턴 보정

`schemas/tokens.v1.json`의 `token.id` 패턴을  
`^token:[a-z0-9]+` → **`^token:[a-z0-9][a-z0-9_-]*`** 로 수정.

이유: TYPO 컬렉션의 `font_family/title` 토큰이 `_` → `-` 변환 후 `token:font-family.title`로 생성되는데, 원래 패턴의 첫 세그먼트 `[a-z0-9]+`는 `-`를 허용하지 않았음. 첫 글자는 여전히 `[a-z0-9]`로 제한, 이후에만 `_-` 허용.

## 실패 및 원인

- **스키마 패턴 실패**: TYPO fixture 첫 validate에서 `/tokens/0/id must match pattern` 오류.  
  원인: `font_family` 이름의 첫 path segment가 `_` → `-` 변환 후 `-`를 포함, 기존 패턴 `[a-z0-9]+`에서 불허.
- **WDS figmaKey 없음**: `verbosity=standard`에서 Figma Variable `key` 필드가 미포함.  
  대응: `v.key || v.id` fallback으로 VariableID 값을 figmaKey로 사용 (안정적 식별자).

## 해결 방법

- 스키마 패턴 `^token:[a-z0-9]+` → `^token:[a-z0-9][a-z0-9_-]*` 로 최소 수정. 기존 token sample들은 모두 PASS 유지.
- figmaKey fallback: `v.key || v.id` — verbosity=full로 재수집 없이 해결.
