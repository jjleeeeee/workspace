## 진행 과정

### 1. v2 스키마 커버율 분석

기존 카탈로그 필드 → v2 섹션 매핑 전체 검토.
- 전체 커버율: ~89% (diagnostics 포함 시 79%)
- 핵심 개선점 확인: spec(디자이너) / figma(MCP 자동) / resolved(AI 생성) 책임 분리 구조가 카탈로그 작성 역할 분담과 일치
- Sprint-1 3개 tool 적합성: `get-ds-component-metadata`는 매우 높음, `list-ds-components`는 높음, `list-ds-tokens`는 tokens.v1.json과 별도 파일 유지 권장

### 2. 이슈 3가지 발견

| 이슈 | 내용 | 심각도 |
|---|---|---|
| `resolved.dimensions` additionalProperties: false | Button 중심 고정 필드로 Badge, Divider 등 다른 컴포넌트 추가 불가 | 필수 수정 |
| `stateTexts` 필드 누락 | ChattingFields·TextFields의 상태별 placeholder 데이터 수용 위치 없음 | 필수 수정 |
| `resolved.tokens` description과 기존 가이드 예시 불일치 | 가이드의 `element→mode→tokenRef+light+dark` 패턴이 v2 description에 미반영 | 권장 수정 |

### 3. 스키마 보완 (schemas/components.v2.json)

1. `resolved.dimensions` → `additionalProperties: true` + 설명 문자열 추가
2. `resolved.stateTexts` 선택 필드 신규 추가 (입력 컴포넌트 placeholder 전용)
3. `resolved.tokens` description에 기존 카탈로그 패턴 예시 추가

### 4. 카탈로그 가이드 업데이트 (cds_component_src/00_catalog_creation_guide.md)

- **Step 1 템플릿** — flat 구조 → v2 5섹션(meta/spec/figma/resolved/diagnostics) 구조로 전면 교체, 섹션별 작성 책임 명시
- **Step 2 필드 참조** — `summary.description` → `spec.identity.description` 등 전체 v2 기준으로 업데이트
- **Step 5 초안 Write** — `resolved.tokens` + `figma.tokenBindings` 이원화 반영
- **Step 6 보완 단계** — `resolved.typescript`, `spec.behavior`, `resolved.stateTexts` 기준으로 업데이트
- **섹션 5 작성 요령** — `tokens` → `resolved.tokens`, `componentProperties` → `figma.props`, `props.typescript` → `resolved.typescript` 예시 코드 교체
- **품질 체크리스트** — 전체 필드명 v2 기준으로 일괄 업데이트

## 발견/결과

### v2 스키마 강점 (v1 대비)

- `figma.states[].diffFromDefault` — 상태별 시각 변경 항목 명시 가능
- `figma.tokenBindings[]` — 역할×축×상태 3중 교차로 토큰 바인딩 정확도 향상
- `diagnostics` 섹션 — 토큰 미해결/하드코딩/상태 누락 등 자동 헬스체크 구조화
- LLM 즉시 사용성 — 한 파일에 spec+figma+resolved 전부 → `get-ds-component-metadata` 응답으로 그대로 사용 가능

### 기존 카탈로그와의 호환성

기존 카탈로그(button, chatting_fields 등)는 v2로 마이그레이션 시 섹션 재분류가 필요하나, 데이터 손실 없음. `resolved.tokens`는 기존 `tokenRef+resolvedLight+resolvedDark` 패턴을 그대로 허용(additionalProperties: true).
