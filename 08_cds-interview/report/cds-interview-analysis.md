# CDS 사용자 인터뷰 심층 분석 리포트

> 인터뷰 기간: 2026.05.06 – 2026.05.14  
> 대상: DS 사용 디자이너 8인 (1:1, 30분)  
> 인터뷰 목적: 디자인시스템(DS)이 실제 업무에서 탐색·의사결정·협업의 기준으로 어떻게 작동하는지 이해

---

## 인터뷰이 개요

|---|---|---|---|
| 문한나 | 어카운트·미디어라이브 | 높음 | 명칭 혼선(배지), 플랫폼별 개발완료 가시성 부족, 사례집 부재 |
| 이수연 | DM·프로필 (주니어) | 높음 | 개발이 CDS V2 무시·AI 우회, 다국어 줄수 제약 |

| 이유솔 | 멤버십 | 높음 | 패턴화의 창의성 저하, 도메인 라이브러리 거버넌스 |

## 가설 검증 결과

### H1. 컴포넌트 탐색 비용이 높아 새로 그리거나 복사함

**신호 강도: ⭐⭐ 조건부**

- 검색 자체는 빠르다는 응답이 우세. 수연 "이제 익숙해져서 그냥 가볍게 검토만 하고 바로 나오는 편", 한샘 "서치가 저는 제일 빨라서 좋다", 소의 "왼쪽 패널에 S 눌러서 바로 찾아진다"
- 본질은 탐색이 아닌 **신뢰성·최신화 비용**: 한나·수연이 기존 화면에서 가져온 후 *CDS 코드 파일을 한 번 더 재확인*. "기존 화면이 업데이트 안 됐을 수 있으니까"
- 신규 컴포넌트 인지 채널 부재: 한샘 "신규로 자꾸 늘어난다 했을 때 찾기가 힘들고 직접 들어가서 봐야"

**핵심 재정의**: H1의 실체는 탐색 속도가 아니라 "이게 최신인가·개발됐나"에 대한 **검증 비용**.

---

### H2. 컴포넌트/레이어 명칭이 개발 용어와 달라 협업 혼선

**신호 강도: ⭐⭐ 부분 지지**

- **디자이너 내부 학습 혼선**이 주된 형태:
  - 한나 "배지를 생각하면 칩과 유사한 형태라 생각했는데 CDS에서는 탭 느낌이잖아요. 막 찾으려고 이렇게 계속 봤던 적은 있고"
  - 한샘 "타이틀 헤더라고 하는게 명칭이 너무 긴 거 같아"
- **개발자와의 명칭 충돌은 생각보다 적음**: 수연 "CDS로 링크를 연결해 놓으면 별말 없이 잘 그대로 해주신다"
- **기획자와의 충돌이 더 큼**: 한샘 "기획은 잘 모르시는 거 같고 이걸 왜 써야 하는지도 '이거 디테치해서 바꾸면 되지 않나요?' 하는데"
- 컬러 토큰 표기 충돌 사례: 소의 "컬러값을 붙여서 두면 개발에서 '어 이거 어떤 컬러인지 모르겠다'고 답변 주신 적 있어서"

---

### H3. 가이드 문서는 있지만 실제 작업 중 활용 부족

**신호 강도: ⭐⭐⭐ 양극화**

- **적극 활용파**: 유솔은 개발자와의 스펙 합의 도구로 사용. "유세이지를 통해서 개발자분들과 컴포넌트 스펙을 확인할 때도 있다", 이중 팝업 등 정의 누락 케이스의 협의 근거로도 사용
- **거의 안 보는 파**: 수연·소의 — 익숙해진 후 CDS 파일에서 바로 확인
- **공통 불만**: 진아 "이건 이거 할 때 사용합니다 — 너무 간단하게만 설명. 어떠한 사용성을 부여하기 위한 장치입니다, 좀 더 자세하고 강의적으로 서술됐으면 좋겠다"
- **사례집 부재**: 한나 "리스트 아이템을 이렇게 썼어요라는 실제 사례집 같은게 있으면 도움이 많이 될 거 같다"
- 대안 행동: 문서 대신 **기존 화면을 레퍼런스로 끌어와 작업**(단비·수연·한나 공통)

---

### H4. 신규 화면/서비스에서 DS가 제약으로 느껴짐

**신호 강도: ⭐⭐⭐⭐ 강한 지지**

- 창규 "신규 작업 시에는 쓸 수 있는 컴포넌트가 부족하여 어려움. 선행 디자인은 일단 시안을 만든 후 DS에 맞춰 워싱"
- 한샘 "버티컬 서비스라 컴포넌트를 쓸 일이 많이는 없었고, 제약이 좀 많이 느껴졌다"
- 진아 "처음부터 '별도로 가자'고 협의했다" — 리스닝파티 독립 라이브러리
- 단비 "그냥 예외처리 하시면 됩니다라고만 답이 오니까, 예외처리가 많이 쌓여도 되는 건가?"
- 반례: 유솔은 JPFC 스펙 변동이 많은 과제에서 오히려 "컴포넌트 활용이 훨씬 효율적"이라며 안전판으로 기능

---

### H5. 시스템 규격 밖 변주는 UX 목적/판단에서 발생

**신호 강도: ⭐⭐⭐⭐ 강한 지지**

변주의 실제 사유가 모두 정당한 UX 근거를 동반:

- **법적 제약**: 한나 "회원가입 화면을 최대한 줄이려고 — 컴포넌트를 다 쓰면 줄이 늘어나서 커스텀해야 될 거 같아요"
- **시각 위계 부족**: 한나 "거주국가 변경 시 리스트아이템으로 썼을 때 타이틀이 강조되지 않아 시각적으로 약하다"
- **다국어/현지화**: 수연 "스페인어가 있어서 두 줄 제약을 해도 세 줄이 될 수 있어요. 차라리 한 줄로 제한하는 게 더 맞다"
- **브랜딩 톤**: 단비 "이벤트 컬러·여백·날카로움 같은 사소한 것" 변주
- **전달력**: 한샘 "0에서 자유롭게 그린 뒤 나중에 교체 — 기획자 간의 이미지가 중요"

---

### H6. DS가 기획/개발 협업에서 공통 언어·설득 근거로 부족

**신호 강도: ⭐⭐⭐ 양가적**

**개발 협업에서 강한 설득 근거로 작동:**
- 한나 "개발 리소스가 더 드는 거라는 논리적 이유 중 하나가 되는 것 같아서 좋아"
- 수연 "CDS로 링크를 연결해 놓으면 별말 없이 잘 그대로 해주신다"
- 소의 "여백이나 컬러 같은 것들 그냥 바로 얘로 해 주세요 하면 되니까 표현할 때 확실히 편해졌다"
- 유솔 "QA에서 속도가 빨라졌다는 느낌"

**기획 협업에서 설득 한계:**
- 한샘 "기획은 잘 모르시는 거 같고 이걸 왜 써야 하는지도 모를 거 같고 '그냥 디테치해서 바꾸면 되지 않나요?'"
- 진아 "컴포넌트 상단 한 줄 설명이 기획자를 설득하는데 중요한 역할을 해줘야 하는데 지금은 너무 간단하게만"

**개발 측 인지 격차 (신규 발견):**
- 수연 "iOS에서 V2를 전부 버리고 한땀한땀 만들어버린 케이스. 일정에 쫓기면 야매로 개발해서 빠르게 넘기시는 경우가 더 많다"

## 워크플로우 패턴 분석

### 시안 시작 패턴

| 유형 | 인터뷰이 | 사유 |
|---|---|---|
| **CDS 먼저 조립** | 이수연·이유솔·손소의·문한나 | 개발 효율, QA 안정성, 일정 버퍼 협상 |
| **자유 시안 → DS 워싱** | 이창규·한샘 | DS가 UX/브랜딩 의도를 좁힌다는 느낌 |
| **처음부터 별도 라이브러리** | 김단비·김진아 | 신규·브랜딩성 강한 도메인 |

**핵심**: 숙련도가 아니라 **도메인 성격**(신규·브랜딩·하이브리드)이 워크플로우 유형을 결정.

### 탐색 진입점 (1차 → 2차)

| 유형 | 주 사용자 |
|---|---|
| 기존 유사 화면 복사 → CDS 코드 파일 재확인 | 문한나·이수연·김단비·한샘 |
| CDS 좌측 패널 직접 검색 | 손소의·한샘·이유솔 |
| 동료/DS팀에 질문 | 김단비·한샘 |
| 챗봇(어둠이) 활용 | 문한나 (단독) |

### 디태치 트리거 6분류

| 유형 | 사례 |
|---|---|
| 물리적 한계 (글자수·너비) | 수연 툴팁, 한나 회원가입 텍스트필드 |
| UX 위계 부족 | 한나 거주국가, 유솔 멤버십 헤더 |
| 하이드/앵커링 오류 | 창규 (이유로 부분 복사 방식 채택) |
| 명칭 못 찾음 | 한나 배지, 한샘 타이틀헤더 |
| 개발 미구현·V1/V2 혼재 | 수연·유솔 |
| 브랜딩/도메인 톤 | 단비·진아 |

## 가설 외 신규 발견 4가지

### 🔴 발견 1: "혼날까봐" — DS팀·디자이너 권력 비대칭

가장 강한 신규 신호. 3명에게서 명시적 자기검열 발언:

- **김단비**: "일단 만들어도 될까? 아 혼날 것 같은데, 정원님한테 혼날 것 같은데"
- **이유솔**: "제 맘대로 만들면 안 될 것 같다는 소심함이 들더라고요"
- **한샘**: "판단력이 없어가지고 자꾸 물어보는 거 같아"

→ DS팀이 **판관/허락자 위치**로 인식됨. 제안 채널의 심리적 안전감이 도구 개선보다 큰 레버.

문한나는 챗봇("어둠이")을 "사람한테 물어보기 부담스러운 영역"을 대체하는 용도로 사용 — 권력 비대칭을 간접 확인.

---

### 🔴 발견 2: 컴포넌트 "개발 상태 가시성" 부재 = 신뢰 손상의 공통 원인

거의 전원이 이 문제로 신뢰 비용을 치름:

- 한나: "V1·V2, iOS·안드로이드 동기화 불투명 → 매번 코드 파일 재확인"
- 수연: "iOS 개발팀이 V2를 전부 버리고 한땀한땀 구현. AI가 CDS 못 잡아내고 야매로 던져놓은 거 그대로 활용"
- 단비: "어디서부터 어디까지 WDS 된 건지 정확하게 내가 모르니까"
- 유솔: "V1/V2 혼재 컴포넌트 경우 문의량이 확실히 있었다"

→ **컴포넌트별 [iOS·Android·Web × V1·V2 × 채택 여부] 상태 매트릭스** 노출이 핵심 수요.

---

### 🔴 발견 3: 도메인 라이브러리 거버넌스 공백

4명(단비·진아·소의·유솔)이 자체 라이브러리 제작 경험. 5개 공백이 공통:

1. **승인·검토 프로세스 부재** — "만들어도 될까?"
2. **CDS 용어·프로퍼티 매핑 가이드 없음** — 진아·유솔
3. **도메인 라이브러리 → CDS 승격 기준 없음** — 소의(인터뷰 중 처음 가이드를 받음)
4. **스펙 변동 대응 속도 기준 없음** — 유솔
5. **도메인 간 컴포넌트 침투 관리 부재** — 진아 카드가 피드·디스커버로 흘러간 사례

---

### 🔴 발견 4: "일관성 vs 디자이너 정체성" 충돌

DS 채택 저항의 본질이 **효율성이 아닌 정체성**에서 옴:

- **한샘** (가장 강함): "다 비슷비슷해 보이는데, 이걸 차이를 두는 게 맞는 걸까 일관성을 가지고 가는 게 맞는 걸까"
- **김단비** (자기억제로 전환): "디자이너 특성상 계속 새로운 거 해볼까 하는 때가 있잖아요. 근데 그런 쓸데없는 고민을 안 하고 본질적인 고민을 하게 해줄 수 있는 거"라며 DS를 자기억제 도구로 긍정 전환
- **이창규**: "누군가가 더 좋은 것을 만들 수 있다고 판단하여 시스템을 깰 수 있다는 관점 — DS를 깨는 게 능력 표현이라는 인식"
- **이유솔**: "디자이너니까 뷰라든지 이런 걸 커스텀하고 싶은 욕심이 존재"하되 DS 우선으로 자기조절

---

### 🟡 추가 발견: 웹/하이브리드 사각지대 — "소외감"

단비가 유일하게 **"소외감", "왕따"** 단어를 직접 사용:
> "WDS 왕따 당하는 거잖아요. 소외감이 들어"

글로벌탭(앱)↔스팟(웹) 토큰 불일치(17→19→21 점프), 베트남 외주 마크업으로 직접 소통 단절까지 겹침. 1명 사례이지만 구조적 문제.

## 협업 다이내믹스

### 기획자
- DS 이해도 가장 낮음. "디테치해서 바꾸면 되지 않나요?"(한샘), "이런 컴포넌트가 있어요 이것도 커스텀이에요 이런 질문을 많이"(한나)
- 디자이너가 DS를 **설득 방패**로 활용: "개발 리소스가 더 든다는 논리적 이유"(한나), "코드와 별다르지 않다면 기획 설득"(유솔)
- 진아: "컴포넌트 상단 한 줄이 기획자를 설득하는 데 중요한 역할을 해줘야 한다"

### 개발자

### 인물별 발견

#### 김단비

1. **"혼날 것 같다" 자기검열 vs DS = 본질적 고민 도구 긍정 재구성 (B)**

   - 종합 평가: "쓸데없는 고민을 덜하게 해 준다 … 본질적인 고민을 하게 해 줄 수 있는 거"
   - 권력 비대칭 정서 부담을 인지적 재해석으로 해소. 죄책감 → 자기억제 장치 미덕화.

2. **"WDS 왕따" 소외감 vs DS 옹호 (B)**
   - "WD 왕따를 당하는 거잖아요 … 자꾸 제가 하는 프로젝트는 WDS를 충실하게 할 수 없는 상황"
   - 동시: 앱 영역 DS 효율 인정. 영역별 분리 인지 + 자기 처지 박탈감 동거.

3. **가이드 정성 작성 vs "자기만족" 자각 (B)**

   - 노동의 의미 자체를 의심하면서도 계속함. 디자이너 자기 노력의 효용 균열.

4. **신규 컴포넌트 제안 회로 인지 vs 두려움 차단 (A)**

   - 제안 채널 존재 인지함에도 정서 장벽으로 차단.

   - 컨셉 차별화 욕구 + 권력 부담 → 변주 욕망 컬러로만 발산.

   - "그때 마음이 아프죠" (디테치 시)

   - 호평(잠재력)과 탐색 brute force 동거. 단비·이수연과 동일 패턴.

5. **V1/V2 헤더 혼란: DS 책임 + 본인 책임 동시 자인 (A)**
   - DS 측 책임: "혼재되어 있으니까 헷갈리지 … 이모지로 다이아몬드 놓치긴 했는데"
   - 본인 측: "업데이트라는 업무가 후순위로 밀리잖아요 … 오랜만에 들어가면 V1이 깨 남아 있으니까"
   - DS-디자이너 공동 부채 자각.

6. **"제가 쓸 땐 편한데 기획자는 모름" (A)**
   - 본인 효율 인정 + 기획자 인지 부재 자인 동시
   - 디자이너만 DS 혜택 누리는 비대칭 명시.
